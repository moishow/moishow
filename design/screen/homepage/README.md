## 1. 페이지 역할

홈은 **그 자체로 완결되는 화면이 아니라**, 흩어진 도메인(crew·user)의 "지금 할 일"과 진입점을 한 화면에 모으는 **집계 + 라우팅 허브**다. 홈의 고유 가치는 **크로스 도메인 집계**(여러 모임의 회비·일정·알림을 한 곳에)에 있고, 깊은 처리는 각 도메인 화면으로 위임한다.

**역할 경계 원칙**
- 홈이 소유: 집계·요약·진입점·글로벌 네비게이션
- 홈이 위임: 실제 처리(송금·가입·정산·정산확인), 상세 조회, 목록 관리

## 4. 핵심 기능 (필수 구현) — 요약

1. 인사 + 오늘 할 일 건수, 벨 알림 진입점
2. 오늘 챙길 일 큐(회비 송금·추가정산·환급수령·(총무)송금확인) + 액션 버튼 → crew 처리
   - 정렬: 잔여시간 오름차순 → 마감없음 최신순(P2) · 상태 갱신: 푸시+재조회+당겨서 새로고침(P4, 폴링 X)
3. 캘린더 스트립(멀티위크 스크롤 + 날짜탭 인라인 필터) + 정모 진입
4. 내 모임 카드(상태 badge) + 모임 홈 진입(D4) + 전체보기(5개+에서만·D5) + 찾기·만들기(탐색 전용)
5. 추천 태그 → 탐색(W-06) · 추천 모임/번개 2섹션(콜드스타트)
6. 하단 5탭 + FAB 만들기(글쓰기/모임/정모·총무/번개)

> 제거: 회비 현황 요약 카드(P3, 마이 이관) · "전체 할 일" 더보기(D3) · 인사 영역 프로필 딥링크(D1)

## 6. 책임 주체 재배치 요약 (홈에서 덜어낼 것)

- **상세·목록·처리는 홈이 소유하지 않는다**: 회비 상세→W-25, 알림 목록→W-26, 정모 처리→crew, 탐색→W-06
- 홈에 **남는 것은 "집계 + 진입점 + 글로벌 네비"** 뿐 — 이 경계를 넘어 홈에 기능을 얹지 않는다
- GAP(모임 홈·내 모임 목록·회비 송금·총무 송금확인 화면)은 **crew/user 도메인이 책임** — 홈은 진입점만 제공

## 7. 명세 확정 — 남은 것은 "제작"뿐

홈페이지 기능 명세는 D1~D8·P1~P4 전 항목 확정으로 **종료**되었다. 남은 작업은 설계 논의가 아니라 구현 대상 화면 제작이다.

- **crew 도메인 제작 대기**: 멤버용 모임 홈(D4)·내 모임 목록(D5)·회비 송금 화면·총무 송금확인 화면·**번개 상세/개설**(P1)
- ✅ **홈페이지 HTML 갱신 완료**: 구 모델·용어(예치/동의·내 포인트·내 동아리·쇼) → 본 문서 확정 스펙(송금 딥링크·모임/정모/피드) 반영. 상단 검색 아이콘→W-06 추가(B5). [`homepage.html`](homepage.html)
- ✅ **홈 빈 상태(콜드스타트) 구현 완료**: 신규 사용자(오늘 챙길 일 0·내 모임 0) → 마스코트 환영 히어로 + 추천 모임/번개 풀노출(D6). [`cold-start.html`](cold-start.html)
- ✅ **FAB 만들기 액션시트(D7) 구현 완료**: 글쓰기/모임 만들기/정모 개설(총무만)/번개. 별도 상태 파일로 분리. [`create-sheet.html`](create-sheet.html)
- ✅ **형식 통일**: 세 홈 화면 모두 다른 W-xx와 동일 형식(흰 배경·단일 폰). 와이어플로우 [`../../../docs/scenario/home/wireflow.html`](../../../docs/scenario/home/wireflow.html)에 반영.
- **crew 도메인 제작 대기**: 멤버용 모임 홈(D4)·내 모임 목록(D5)·회비 송금 화면·총무 송금확인 화면·**번개 상세/개설**(P1)
- 다음 단계: crew/user 도메인 시나리오 규정(Step 1)
