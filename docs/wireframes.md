## 개요

정본 와이어프레임은 **하이파이 와이어플로우 보드**다. 8개 시나리오(S1~S8)를 레인으로 펼치고, **실제 v2 화면**(`design/system/screens/W-xx.html`)을 흐름 안에서 화살표·분기로 연결한다. "시나리오에 따라 페이지가 어떻게 이어지는지"를 실제 화면 그대로 본다(중복 페이지 의도적 허용).

- 각 노드는 로파이 스켈레톤이 아니라 **실제 v2 디자인시스템 화면**을 축소 삽입한 것.
- 각 화면(`design/system/screens/*.html`, 40개)은 **단독 실행 가능** — 화면의 단일 소스이자 목업/개발 핸드오프에 재사용.
- 색 출입 권한은 canonical `home.html` 기준(클럽 아바타·대기 배지는 코랄 = 클럽 정체성 허용).
- 참고 로파이: `design/spec/Moisho Spec.html`. (v1 프로토타입·토큰은 제거됨 — 정본은 `design/system/` v2)

## 시나리오 플로우 (S1~S8)

### S1 · 소셜 로그인 + KYC + 지갑 개설 — 👤 부원·총무 · **Must**

신규 진입 → 3초 로그인 → (신규)본인인증 → 지갑 자동 개설 → 홈. 성공 기준: **verified** + 지갑 생성.

```
⟦시작⟧ → [W-01 온보딩·SNS 로그인] →(로그인)→ ◇ 신규 가입?
   ├─[신규]→ [W-03 KYC 인증] →(verified)→ [W-04 지갑 개설] → [W-05 홈] → ⟦홈 진입⟧
   └─[기존]────────────────────────────────────────────────→ ⤳ W-05 홈 (위와 동일)
```

### S2 · 모임 탐색 + 가입 신청(폼빌더) — 👤 부원 · **Must**

관심사 태그 탐색 → 클럽 상세(안심예산안 프리뷰) → 가입 신청서 → 총무 승인 → Membership + 알림.

```
⟦시작⟧ → [W-05 홈] →(탐색)→ [W-06 모임 탐색] → [W-07 클럽 상세]
   →(신청)→ [W-08 가입 신청서] → [W-09 신청 완료]
   →(총무 승인)→ [W-26 알림] → ⟦멤버십 생성 · 홈⟧
```

### S3 · 차수 개설·예치 (에스크로 OPEN) + 만남 전 취소 — 👤 총무·부원 · **Must ⭐**

총무가 차수 개설 → 부원이 차수 상세(달성 게이지)에서 예치(부족분 카카오페이 충전) → OPEN. 만남시각 전이면 자유 취소 → 즉시 환불(총무 출금 잠김 유지). 예치는 멱등.

```
⟦시작⟧ →(총무 개설)→ [W-10 모임 만들기(총무)] →(부원 참여)→ [W-11 차수 상세]
   →(예치)→ [W-12 예치] → ◇ 잔액 충분?
      ├─[부족] → [W-13 충전] →(충전 후)→ [W-14 예치 완료]
      └─[충분] ─────────────────────────→ [W-14 예치 완료]
   → ◇ 만남 전 취소?
      ├─[취소] → [W-15 취소·환불] → ⟦즉시 환불⟧
      └─[유지] →(만남시각 도달)→ ⤳ S4 — 락·동의
```

### S4 · 만남시각 락 → 10분 동의 창 / 노쇼 → 총무 출금 — 👤 총무 + 부원 · **Must ⭐**

락 → 총무 동의 요청(락 스냅샷 전원 푸시, 10분 창) → 부원 동의(거절 없음) → 마감(미응답=노쇼) → 동의자≥1이면 총무 출금(정산 타이머), 0명이면 VOIDED 전원 환불.

```
⟦만남시각 도달⟧ → [W-11 차수 상세(락)] →(총무)→ [W-TR 총무 대시보드]
   →(동의 요청 발송)→ [W-26 알림(전원)] →(부원)→ [W-16 동의 10분창]
   → [W-17 동의 마감 결과] → ◇ 동의자 ≥ 1?
      ├─[예] → [W-18 총무 출금] →(출금)→ ⤳ S5 — 차수 종료·정산
      └─[0명] ───────────────────────→ ⟦VOIDED · 전원 환불⟧
```

> 미응답 = **노쇼**: 예치금×노쇼율(주최자 설정) 몰수 → 에스크로 잔류 + 잔여 환불.

### S5 · 차수 종료 OCR → 잉여 환급 / 적자 추가정산 → 자동 정산 — 👤 총무 → 부원 · **Must ⭐**

차수 종료(영수증 OCR 강제 매핑·실지출 확정) → 잉여면 잔액 반납·균등 환급 / 적자면 동의자 균등 추가징수(미납=프로필 표시+지속 푸시) → 자동 정산(DONE) → 현금화. **불변식: DONE 시 잔액 0.**

```
⟦출금 완료⟧ →(차수 종료하기)→ [W-19 차수 종료 OCR] → ◇ 잉여 / 적자?
   ├─[잉여] →(자동 균등환급)→ [W-23 정산 완료] → [W-25 지갑/마이] →(현금화)→ [W-24 현금화] → ⟦완료⟧
   └─[적자] → [W-21 추가정산 요청] →(부원)→ [W-22 추가정산 납부] → ◇ 납부?
         ├─[납부] → ⤳ W-23 정산 완료
         └─[미납] → ⤳ S8 — 미납 추적
```

### S6 · 소셜 — 쇼 피드 / DM — 👤 부원 · *Should*

쇼 글·사진 작성/반응/신고(→ 모더레이션). DM 대화. 운영 문의 → CSTicket. 쇼츠(숏폼) 제외 — 글/사진 피드만.

```
⟦시작⟧ → [W-05 홈] → 분기
   ├─[쇼 탭] → [W-27 쇼 피드] → 분기
   │     ├─[글 열기] → [W-33 쇼 상세] → 분기
   │     │      ├─[댓글·좋아요] → ⟦반응⟧
   │     │      └─[신고] → [W-31 신고] → ⤳ S7 — 모더레이션(W-44)
   │     └─[작성] → [W-28 글 작성] → ⟦게시⟧
   ├─[DM 탭] → [W-29 DM 목록] → [W-30 대화방] → ⟦대화⟧
   └─[문의] → [W-32 고객센터] →(CSTicket)→ ⤳ S7 — CS(W-45)
```

### S7 · 운영자 — 심사·지급 감독·대사·모더레이션 — 🛠 운영자 · **Must**

대시보드에서 분기: 모임/KYC 심사, 에스크로·원장 감사(append-only·CSV), 카카오페이 충전 대사, 신고 모더레이션, CS, 푸시, 리포트. 민감 작업은 AuditLog.

```
⟦로그인⟧ → [W-40 대시보드] → 분기
   ├─[심사]   → [W-41 심사 큐]
   ├─[감사]   → [W-42 원장 감사]
   ├─[대사]   → [W-43 충전 대사]
   ├─[모더]   → [W-44 모더레이션]
   ├─[CS]     → [W-45 CS 티켓]
   ├─[푸시]   → [W-46 푸시 발송]
   └─[리포트] → [W-47 리포트]
```

### S8 · 엣지 — 노쇼 · 추가징수 미납 · 충전 지연 · 환불 분쟁 — 👤 부원·운영자 · *Should*

예외 4종. **강제 출금 없음** — 미납은 프로필 표시 + 지속 푸시로만. 충전 지연은 안내 + 재시도/DLQ. 모두 멱등 전제.

```
[노쇼]      [W-16 동의창(미응답)] →(페널티 몰수)→ [W-25 지갑] → ⟦잔여 환급⟧
[미납]      [W-22 추가정산(미납)] → [W-25 프로필] →(지속 푸시)→ [W-26 알림] → ⟦미납 추적 지속⟧
[충전 지연] [W-13 충전(지연)] → [W-26 안내 알림] → ⟦충전 확정⟧
            └ 재시도 · DLQ · 멱등(partner_order_id) — 이중 확정 금지
[환불 분쟁] [W-32 고객센터] →(CSTicket)→ [W-45 CS(어드민)] → ⟦처리⟧
```
