# 알림 — 알림 (횡단)

> 상위: [모임 도메인 시나리오 인덱스](../../README.md) · 원 시나리오: X1 `[검증]` ⚠경계

## 개요
모임 도메인이 발생시키는 알림 트리거의 정의와 딥링크 목적지. **알림 인프라·설정·목록 화면의 소유는 공용/회원 도메인과 조율 필요**(예경님 스펙도 알림센터를 user로 분류) — 우리는 트리거 정의를 소유한다.

## 행위자
전 역할(수신) · 시스템(발송)

## 트리거 → 딥링크 매핑

| 트리거 | 수신자 | 딥링크 목적지 |
|---|---|---|
| 공지 등록 | 부원 전체 | 게시글 상세(F2) |
| 정산 요청 생성 | 참여자 | 정산 상세 |
| 납부 기한 임박·리마인드 | 미납 참여자 | 정산 상세 |
| 셀프 신고 발생 | 주최자 | 납부 현황판 |
| 납부 확인/반려 | 해당 참여자 | 정산 상세 |
| 정산 수정·취소·환불 | 참여자 | 정산 상세 |
| 이의 제기/정정 결과 | 주최자/참여자 | 정산 상세 |
| 정산 마감 | 참여자(+부원 선택) | 정산 아카이브 |
| 가입 승인/거절 | 신청자 | 모임 홈 / 가입 신청 화면(거절) |
| 가입 신청 발생 | 모임장·관리자 | 가입 신청 목록 |
| 이벤트 취소/변경 | 참여자 | 정모/번개 상세 |
| 정모·번개 개설 | 부원(내 모임)/팔로워 | 정모/번개 상세 |
| 신고 접수/처리 | 관리자/신고자 | 신고함 / 게시글 |
| 모임장 승계 | 대상자·부원 | 모임 홈 |
| 강퇴 | 대상자 | (안내 화면) |


## 분기 / 엣지
- ⚠ 알림 목록 화면 소유(회원 vs 공용) 팀 조율 · 알림 설정(on/off)은 회원 도메인
- 그룹핑(같은 글 좋아요 N건) · 발송 빈도 제한(리마인드)은 2차 검증

## 검증 이력
`[검증]` 1차 검증 편입 · v6~7에서 트리거 목록 갱신(납부 확인·정정·환불 추가)
