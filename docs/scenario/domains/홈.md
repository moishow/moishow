# 홈 도메인 — 시나리오 규정 (#10 결정 반영 초안)

> v2 · 2026-07-13 · 원저자: **예경(kyung 브랜치 `docs/scenario/home/`)** · 재구조화·#10 반영: 해찬
> ⚠ **오너 확정 대기** — #10 팀회의 결정(2026-07-13, 본문 `#10-N 확정` 마커)을 반영한 초안. 최종 확정 권한은 홈 도메인 오너(예경)에게 있음.

## 1. 도메인 정의
- **역할**: 진입·대시보드·라우팅 허브. 깊은 작업(회비 송금·가입·정산)은 각 도메인(모임/회원)으로 **핸드오프**.
- **제품 전제**: 모바일 앱 — 전원 로그인 상태(모임 도메인 스펙과 공유).
- **행위자**: `부원(member)` / `모임장·관리자(운영 todo 노출)`.
- **정산 모델(#10-1 확정)**: 송금 딥링크 → [송금완료] → 확인대기 → 주최자 대조 → 완료/미송금 (모임 셀프 신고+주최자 확인과 동일 ✅). **정산 방식은 주최자가 개설 시 선택** — 사후 정산 / 사전 걷기(선납→차액 환급·추가정산). 홈 큐·알림은 이벤트 방식에 따라 항목이 달라짐.

## 2. 도메인 경계
- **→ 모임**: 내 모임 카드·이벤트·회비 액션의 목적지. **탐색 소유(#10-3 확정)=홈** — 모임·번개 탐색·검색은 홈이 소유(모임 상세로 핸드오프).
- **→ 회원**: 마이(돈 **기록** 열람) · DM · 알림 센터. **돈의 집(#10-6 확정 — 하이브리드)**: 돈 액션=홈 · 돈 기록=마이.
- **번개 소속(#10-2 확정)**: 번개=모임 소속. 무소속 번개 폐기 — 비부원 참여는 공개 번개가 커버.

## 3. 시나리오 (상세)

### 홈_대시보드 — 진입·개관 인덱스

**개요**
앱 진입/재진입 시 내 상태를 한 화면으로 파악하고 다음 행동으로 분기. 홈은 "조회" 시나리오라 자체 종점이 없고 모든 실질 행동을 다른 시나리오에 위임한다(원저자 노트).

#### 대시보드_열람 — 홈 스캔·분기

**행위자**
부원 전원 (모임장·관리자는 운영 todo 추가 노출 — RBAC 분기)

**트리거 / 진입**
온보딩 완료 후 자동 진입 · 탭바 `홈` · 뒤로가기 누적 복귀

**User Flow**
1. 홈 로드 → 인사 + "오늘 챙길 일 N건" 요약
2. 오늘 챙길 일 → 다가오는 일정 → 내 모임 → 추천 순 세로 스캔
3. 처리할 항목 탭 → 해당 시나리오로 분기

**분기 / 엣지**
- 홈 재선택 시 스크롤·섹션 상태 유지, 데이터만 재조회(A3·P4 원저자 확정)
- 회비 현황 요약은 홈에서 제거(P3) — 마이(회원)로 이관됨

#### 빈_상태_온보딩 — 신규 사용자 콜드스타트

**행위자**
신규 사용자(모임 0 · todo 0)

**트리거 / 진입**
온보딩 직후 첫 홈 진입

**User Flow**
1. 오늘 챙길 일 0건 · 내 모임 0개 → 빈 상태 표시
2. 추천 모임/번개 카드 노출(D6: 콜드스타트에서만) → 탐색으로 유도
3. 첫 모임 가입/개설 후 일반 대시보드로 전환

**분기 / 엣지**
- (논의·원저자) 빈 상태를 별도 시나리오로 뺄지 엣지로 둘지 → 본 재구조화에서 하위 행위로 승격(제안)
- 모임 스펙 빈_상태 카탈로그(공통/빈_상태)와 CTA 톤 일관성 필요

### 오늘_챙길_일 — 회비 액션 큐 인덱스

**개요**
마감 임박 회비 액션을 큐로 모아 즉시 처리로 핸드오프. 원저자 확정: 정렬 P2(마감 잔여시간 오름차순→마감 없음 최신순), 갱신 P4(푸시·포그라운드 재조회·당겨서 새로고침, 폴링 없음).

#### 회비_송금_진입 — 송금 카드 처리

**행위자**
정산 참여자(부원 · 모임장·관리자 자기 몫)

**트리거 / 진입**
홈 todo 카드 `송금` (예: "회비 · 1인 52,000원 · 마감 D-1")

**User Flow**
1. 카드 탭 → 이벤트 상세 경유(금액·수취 확인 — 원저자 권장) → 납부 화면
2. 송금 딥링크 → 실이체 → 복귀 → [송금완료] 신고 → `확인대기`
3. 주최자 확인 → 완료 / 미송금 반려 → 재송금 알림 → 재진입

**분기 / 엣지**
- 이 흐름은 모임 스펙 회비_납부(납부_실행)와 **동일 모델(수렴 ✅)** — 화면 구현 시 단일 화면 공유
- 마감 경과 → 카드 상태 전환(미납) · 중복 송금은 앱이 못 막음(주최자 대조 책임 — 원저자 노트)

#### 추가정산_송금_진입 — 부족분 추가 납부 (사전 걷기 이벤트)

**개요**
사전 걷기 이벤트에서 선납액보다 실지출이 크면 주최자가 추가정산을 요청 → 부원 추가 송금. **사후 정산 이벤트에는 없음**(#10-1 방식별 분기).

**User Flow**
1. `추가정산 송금` 카드 탭 → 추가정산 납부 화면(부족분·근거)
2. 송금 딥링크 → [송금완료] → 확인대기 → 주최자 확인 → 완료

**분기 / 엣지**
- 사후 정산 이벤트에는 이 카드가 뜨지 않음 · 미납 시 리마인드(모임 리마인드_발송)

#### 환급_수령확인 — 잉여 환급 수령 (사전 걷기 이벤트)

**개요**
사전 걷기 이벤트에서 선납 총액이 실지출보다 크면 주최자가 잉여를 역송금 → 부원이 [수령확인]. 노쇼자는 페널티(설정 시)만큼 차감 후 나머지 환급 — 페널티 수입은 실참여자 분담 경감에 쓰임.

**User Flow**
1. `환급 수령확인` 카드 탭 → 환급 내역 확인(선납−실지출 차액)
2. 실수령 확인 → [수령확인] → 정산 완료 기록 반영

**분기 / 엣지**
- 마감 없음 항목이라 큐 하단 배치(P2) · 수령확인 지연 → 주최자 재알림
- 사후 정산 이벤트에는 없음 · 정산 정정(감액) 시의 차액 환불도 이 흐름 재사용

#### 큐_정렬_갱신 — 정렬·상태 갱신 규칙

**규칙 (원저자 확정 보존)**
- **정렬(P2)**: ① 마감 있는 항목(잔여시간 오름차순) → ② 마감 없음(생성 최신순), 동점 시 금액 큰 순
- **갱신(P4·폴링 미사용)**: 푸시 수신 시 배지·목록 갱신 · 앱 포그라운드 복귀 재조회 · 당겨서 새로고침. 주최자 확인이 분~시간 단위라 실시간 불필요

**분기 / 엣지**
- **큐 소스(#10-1 확정)**: {회비 송금(D-N)} 공통 · {추가정산·환급}은 **사전 걷기 이벤트에서만** 노출 · 사후 정산 이벤트는 회비 송금만

### 다가오는_일정 — 시간 축 뷰 인덱스

**개요**
캘린더 스트립 + 다가오는 일정 리스트로 "언제 무엇을"을 확인하고 이벤트 상세로 진입. 원저자 확정: 전체 캘린더 화면 스코프 제외(스트립 좌우 스크롤+인라인 필터로 대체).

#### 캘린더_탐색 — 주간 스트립·인라인 필터

**행위자**
부원 전원

**User Flow**
1. 캘린더 스트립 좌우 스크롤(주 단위, 점=일정 있음) — 월 경계 넘으면 상단 월 텍스트 자동 갱신
2. 날짜 탭 → 하단 리스트 해당 날짜로 인라인 필터(페이지 이동 없음)
3. 일정 없는 날 탭 → "일정 없음" 빈 상태로 리스트 대체

**분기 / 엣지**
- 홈 재선택 시 선택 날짜·필터 보존(A3)
- (논의) 회원 도메인 마이에도 주간 캘린더 존재 — 홈/마이 일정 노출 중복 정리(민준 스펙도 동일 지적)

#### 일정_상세_진입 — 이벤트로 핸드오프

**User Flow**
1. 이벤트 행(정모/번개 배지) 탭 → 이벤트 상세(모임 도메인)
2. 종료된 이벤트 → 정산 단계 표시로 전환(오늘_챙길_일과 접점)

**분기 / 엣지**
- 정모·번개 진입 화면 동일, 생성 경로만 다름(원저자 노트 — 모임 스펙과 일치 ✅)
- (논의·원저자) 일정 소스가 이벤트 시각만인지 별도 이벤트 개념인지 — 데이터 모델 확인

### 알림_확인 — 알림 센터 진입 인덱스

**개요**
벨 아이콘(빨간 점=미확인) → 알림 센터 → 원인 화면 딥링크. 원저자는 알림 센터를 회원 도메인 화면으로 분류(알림 설정도 회원) — 모임 스펙의 "알림 소유 조율" 항목과 연결, 방향은 수렴 중.

#### 알림_열람_딥링크 — 목록·이동

**User Flow**
1. 홈 벨 탭(점등=미확인) 또는 푸시 탭 → 알림 센터
2. 목록 스캔(가입 승인·송금 확인/반려·정산·환급(사전)·독촉·DM·신고 결과)
3. 항목 탭 → 원인 화면 딥링크 → 처리 후 복귀 → 읽음 처리·점 소거

**분기 / 엣지**
- 알림 종류→목적지 매핑은 모임 스펙 공통/알림의 매핑 표와 **통합 필요**(중복 정의 방지, 논의)
- (#10-1 확정) 환급·추가정산 알림은 **사전 걷기 이벤트에서 노출** · 사후 정산 이벤트엔 없음

#### 딥링크_폴백 — 만료 대상 처리

**User Flow**
1. 마감 지난 이벤트 알림 탭 → 원인 화면 대신 **정산 결과로 폴백**(원저자안)
2. 삭제·해산된 대상 → 안내 후 알림 센터 잔류

**분기 / 엣지**
- **폴백 목적지 우선순위(리뷰 2026-07-13 확정)**: 정산 결과 → 지난 이벤트 → 모임 홈

### 모임_번개_탐색 — 발견·가입 진입 인덱스

**개요**
추천 태그·추천 모임·추천 번개에서 새 모임/번개를 발견해 가입·참여로 잇는 획득 경로. 원저자 확정: P1(모임/번개 섹션 분리) · D6(콜드스타트에서만 추천 카드) · D8(개설은 FAB 단일화, 찾기·만들기=탐색 전용) · A1(가입 신청서 항상 경유).

#### 태그_탐색 — 필터 기반 발견

**User Flow**
1. 태그 chip(내 주변·당장 가입 가능·이번 주 활동·관심사 맞춤) 탭 → 탐색 목록(필터 적용)
2. 모임 선택 → 모임 상세(비회원 뷰) → 가입 신청 → 승인 알림 → 내 모임 반영

**분기 / 엣지**
- **가입 방식(#10-5 확정)**: 즉시 가입/승인제 이원화(모임 스펙) 채택. A1("신청서 항상 경유")은 **승인제에만** 적용 — 즉시 가입 모임은 탭 한 번으로 가입(스킵)
- 추천 로직 소스 = 온보딩 관심 카테고리+위치(원저자 노트, 데이터 의존)

#### 키워드_검색 — 검색창 발견 (신설 · 리뷰 2026-07-13)

**User Flow**
1. 탐색 상단 검색창 → 모임 이름·키워드 입력 → 결과 목록(태그 필터와 병행)
2. 모임 선택 → 모임 상세(비회원 뷰) → 가입 흐름(태그_탐색과 동일)

**분기 / 엣지**
- 크리틱 ① "검색창·결과 부재" 해소 — 태그·추천만으로 못 찾던 특정 모임을 직접 검색
- 결과 없음 → 빈 상태 + 추천/태그 유도 · 이미 가입한 모임은 결과에 `가입됨` 표시

#### 추천_모임_가입_진입 — 카드 → 상세 경유

**User Flow**
1. 추천 모임 카드 `가입` 탭 → **모임 상세 반드시 경유**(예산·정보 사전 확인 — 원저자 권장)
2. 신청서 작성 → 대기 → 승인/반려 알림

**분기 / 엣지**
- 이미 가입한 모임은 추천에서 제외(중복 방지 규칙 필요 — 원저자 논의)
- 상세 경유 이후 흐름은 모임 도메인 모임_가입과 동일(수렴 ✅)

#### 추천_번개_참여 — 공개 번개 발견·참여

**User Flow**
1. 추천 번개 카드 탭 → 번개 상세(공개 번개 — 모임 소속)
2. 참여 신청 → (회비 있으면) 종료 후 정산 참여(방식=주최자 선택)
3. 참여 후 모임 상세(비회원 뷰) 노출 = 가입 유입 경로

**분기 / 엣지**
- **(#10-2 확정)** 번개=모임 소속. 홈은 공개 번개 발견·핸드오프만, 흐름은 모임 공개_번개_참여와 동일(수렴 ✅)

#### 콜드스타트_추천 — 노출 조건 규칙

**규칙 (보존)**
- 추천 모임/번개 **카드**는 내 모임 0개인 신규 사용자에게만 노출
- 모임 보유 후에는 **태그 씨앗만** 유지(카드 제거)

**분기 / 엣지**
- (논의) 모임을 전부 탈퇴해 다시 0개가 되면 카드 재노출 여부

### 글로벌_내비 — 탭바·FAB 라우팅 인덱스

**개요**
탭바(홈·피드·FAB·DM·마이)와 내 모임 블록으로 전 도메인 이동. 홈이 루트 내비게이션 소유. 원저자 확정: D7(FAB=만들기 허브, 정모 개설만 RBAC) · D5(전체보기 5개+) · A3(상태 보존).

#### 탭바_라우팅 — 도메인 루트 이동

**User Flow**
1. 탭 선택 → 피드(피드 도메인) / DM·마이(회원 도메인) / 홈
2. 홈 재선택: 스크롤·상태 유지+데이터 재조회(A3·P4) · 이미 홈에서 재탭 → 최상단 리셋

**분기 / 엣지**
- 탭바 구성(DM 탭·FAB 포함 5요소)은 모임 스펙 §1(홈/피드/회원 3도메인)과 표기 차이 — 도메인은 3개, 탭은 5개로 정합 자체는 성립(참고)

#### FAB_만들기 — 만들기 액션시트

**User Flow**
1. FAB `+` → 액션시트: 글쓰기 / 모임 만들기 / 정모 개설 / 번개
2. **글쓰기 선택 시 → 대상 선택(리뷰 2026-07-13)**: **모임 기준**(어느 모임 게시판에 쓸지) / **개인 기준**(개인 피드 글) → 피드 F3로 핸드오프
3. 그 외 선택 → 각 생성 플로우로 핸드오프

**분기 / 엣지**
- **RBAC(D7)**: 정모 개설은 권한자만 노출 — **모임장·관리자**(v10 역할 모델)
- 개설 진입점은 FAB로 단일화(D8) — 찾기·만들기는 탐색 전용
- **글쓰기 대상 이원화(리뷰 2026-07-13)**: 모임 게시글(모임 게시판) vs 개인 게시글(개인 피드) — 피드 도메인 F3와 연동(모임 선택 단계 포함)

#### 내_모임_진입 — 카드 → 모임 홈

**User Flow**
1. 내 모임 카드 탭 → 모임 홈(모임 도메인)
2. `전체보기` → 내 모임 목록(소속 5개+에서만 노출 — D5)

**분기 / 엣지**
- 카드 배지(정산 진행 등) 따라 랜딩 위치 분기 가능성(원저자 논의) — 모임 스펙은 항상 모임 홈 랜딩, 정산은 홈 내 진입(정합 제안)
- 원저자 GAP "멤버용 모임 홈"은 모임 스펙 모임_홈으로 **해소됨** ✅

### 운영_진입 — 관리자 todo·핸드오프 인덱스

**개요**
같은 홈 화면에서 권한자(모임장·관리자)에게만 운영 todo를 추가 노출(RBAC 분기, D2: 별도 메뉴 없음). 처리 자체는 모임 도메인으로 핸드오프.

#### 운영_할일_진입 — 운영 todo 처리 진입

**User Flow (원저자안 보존)**
1. 홈 todo의 권한자 전용 카드: "송금 확인 N건" · "미송금 추적" · "정모 종료·정산 필요"
2. 탭 → 납부 현황판/정산 생성으로 핸드오프(모임 도메인)

**분기 / 엣지**
- 송금 확인·미송금 추적은 모임 스펙 납부_추적과 **수렴** ✅
- **정산 진입(#10-1)**: **정산 만들기(실지출+영수증)**가 기본(모임 정산_생성). 사전 걷기 이벤트면 정산 후 환급/추가정산 카드가 발생. (OCR 자동 정산은 MVP 제외)
- 다중 모임 권한자: todo에 모임 컨텍스트 표기 필요(원저자 노트)
- 역할 표기: **모임장·관리자**(v10 · "총무" 표기 폐기, 리뷰 2026-07-13)

#### 모임_카드_관리_진입 — 모임 홈 경유 원칙

**규칙 (보존)**
- 권한자 todo 탭 → **모임 홈 경유** → 관리 진입 (홈에서 관리 화면 직행 금지)
- 근거: 다중 모임 권한자가 컨텍스트 없이 직행하면 어느 모임인지 모호

**분기 / 엣지**
- 모임 스펙 운영_관리(허브+2하위)와 자연 연결 ✅ — 홈은 문 앞까지만
