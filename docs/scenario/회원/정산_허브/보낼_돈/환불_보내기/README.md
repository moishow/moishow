# 환불_보내기 — 주최자→참여자 송금

> 상위: [보낼_돈](../README.md)

## User Flow
1. 주최자 "보낼 돈"에 `환불 요청` 항목 노출
2. 송금 상세 재사용 → 카카오페이 송금 → 송금 완료 체크
3. 받는 쪽(참여자) 입금 확인 → `입금 확인` 기록

## 분기 / 엣지
- 발생 원천: 원저자안=참가 취소 환불 / 모임 스펙=정산 정정·취소에 따른 차액 환불 — **⚠발생 조건은 #10 종속, 송금 흐름 자체는 동일(수렴)**
- 주최자가 미루면 강제 수단 없음 → `환불 대기` 지속+독촉 알림만(원저자 노트, 모임 철학과 일치)
