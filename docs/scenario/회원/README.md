# 회원 도메인 — 시나리오 규정 (제안본)

> v1-제안 · 2026-07-11 · 원저자: **민준(mypage 브랜치 `docs/scenario/user/`)** · 재구조화: 해찬(scenario-spec 방법론)
> ⚠ **이 문서는 제안본** — 원저자 결정을 보존하고 구조만 이관. 확정 권한은 회원 도메인 오너에게 있음.

## 1. 도메인 정의
- **역할**: 마이(프로필·모임 레벨) + 정산 허브(보낼/받을 돈 요약·실행) + 소셜(팔로우) + 설정. 원저자 노트: "마이 = 소셜 프로필 + 정산 허브의 결합"(결합 타당성은 원저자 검증 항목).
- **원저자 모델 전제**: "카카오페이 기반 회원 간 직접 송금 + 송금 여부 직접 체크(포인트·에스크로 없음)" — 모임 스펙과 **수렴** ✅. 상태 3단(미송금→확인 대기→입금 확인)도 일치 ✅.
- **경계(원저자 명시)**: 상세 송금·정산 처리는 모임 도메인 화면으로 이동, 마이는 요약·진입점만. 내 모임 목록은 홈 소유(중복 배치 안 함) ✅.

## 2. 도메인 경계 ⚠플래그
- **⚠충돌(#10)**: 받을_돈의 "환급 받기"·보낼_돈의 "추가 정산"은 사전 걷기 모델 전제 — 모임 스펙(사후 정산)과 충돌.
- **용어**: "차수" 표기 잔존(일정·정산 내역) → 팀 정본 "정모"로 치환 필요(경미).
- **계좌 등록·인증**: 실계좌 인증 외부 API 전제 — 포인트용 KYC 폐기와는 별개 사안이나, 필요성 자체 재확인 권장(카카오페이 딥링크만 쓰면 계좌 등록이 필수인지 — 논의).
- **모임 레벨(신용등급)**: 점수 산정은 정산 도메인 이벤트가 입력(원저자 명시) — 모임 도메인과 이벤트 스키마 협의 필요.

## 4. 검증 로그

**리뷰 결정 1(2026-07-12, 해찬 리뷰 — 팀 제안)**: **돈 현황의 집 = 홈**. 보낼/받을 돈 요약·진입은 홈으로 이동 제안(예경 P3 "회비 현황→마이" 번복 — #10 안건 추가 필요). 마이 = 프로필·소셜·정산 기록(개인 아카이브)·설정. 정산_허브 중 보낼_돈·받을_돈의 진입점은 홈, 정산_내역은 마이 유지. 근거: 제품 핵심(회비 투명성)이 2뎁스로 밀리는 것 방지 + 홈 todo와 마이 보낼 돈의 중복 제거.

**리뷰 결정 2(2026-07-12)**: 모임_레벨 **원안 유지**(점수·캐릭터 — 민준 설계 존중, 점수 구간·증감 정책은 정산 도메인이 확정 후 전달) + **공개 범위 = 공개**(상대 프로필에도 레벨 표시. 신규는 '이력 없음'으로 구분해 낙인 방지 — "제재보다 기록 노출" 원칙).

**리뷰 결정 3·4(2026-07-12)**: 실명 = **본인에게만 조회**(타인에겐 닉네임만 · 정산 명단의 실명 필요 여부는 팀회의 후보) · 팔로워/팔로잉 목록 = **전체 공개**(비공개 계정은 MVP 제외).

**리뷰 결정 5·6(2026-07-12)**: 보낼_돈·받을_돈 → **모임 도메인 정산으로 통합**(흐름 단일 정의, 회원엔 포인터만 — 3중 정의 해소) · 정산_내역 = **회원 유지, 영구 보관, CSV 내보내기 포함**(총무 인수인계·외부 장부용).

**리뷰 결정 7(2026-07-12)**: 일정 = **홈 단일화**(예경 스트립+인라인 필터안이 정본, 마이 주간 캘린더 제거 — 원저자 양측의 "중복 정리 필요" 메모 해소). 역할 정리: **홈=현황·행동(돈·일정·todo), 마이=정체성·기록·설정.**

**리뷰 결정 8~10(2026-07-12)**: 계좌 등록 = **예금주 인증 유지**(원안 존중 — 오송금 방지) · 정산 필수 알림(송금 요청·입금 확인·반려) = **끌 수 없음 고정 + 사유 표시** · 탈퇴 차단 = **내가 주최한 미마감 정산만**(이관·마감 후 가능), 참여자 미납·확인 대기는 차단 안 함(기록 실명 유지로 충분 — 교착 원천 제거).

**리뷰 결정 11(2026-07-12) — 온보딩 신설(공동 결정)**: 로그인 = 카카오·구글·애플 + 신규 가입 시 전화번호 인증 · 스텝 = 로그인→닉네임→관심사→홈 · 랜딩 = 홈 빈 상태(추천 카드, 예경 D6 연결).

**리뷰 결정 12(2026-07-12) — 알림_센터 신설(공동 결정)**: 소유=회원 확정 · 보관 30일 · 개별/전체 읽음 + 유사 그룹핑 · 단일 목록+종류 배지(탭 분리 없음) · 트리거 정의는 각 도메인 매핑 표 참조(3원 정의 해소).

**리뷰 결정 13(2026-07-12) — 채팅 신설(공동 결정)**: 기존 DM 대체 — **이벤트 채팅=모임 도메인**(자동 생성·입장), **개인 채팅=회원**(1:1 제한 없음, 차단이 안전장치 — ⚠차단·신고는 팀회의 잔여). 목록은 통합 표시.
- 원저자 결정 보존: 마이=요약 허브(상세는 모임 도메인 재사용) · 환불 전용 페이지 없음(송금 상세 재사용) · 모임 레벨=표시 전용(점수 정책은 정산 소관) · 설정=톱니바퀴 진입.
- **크리틱(2026-07-11, 해찬) — 누락 시나리오 후보 4+2**: ① **온보딩·로그인·계정 생성 부재**(예경 분류상 user 소유, 로그아웃 복귀 목적지인데 시나리오 없음 — 최대 구멍) ② **DM**(탭 존재·W-29/30=user 분류인데 부재) ③ **알림 센터 화면**(W-26=user 분류, 홈은 진입만 — 소유자 시나리오 부재) ④ **사용자 차단·신고(사람)·CS 문의**(모임·피드가 회원으로 넘긴 경계의 수신부 부재). 엣지: 분쟁 중 탈퇴 교착(확인 대기 잠김이면 영원히 탈퇴 불가 순환) · 상대 프로필의 모임 레벨 공개 범위. **확정은 오너(민준) 몫.**
- ⚠충돌은 #10로 집약. 재정리 신규 (논의)는 각 문서 분기/엣지 — 확정은 오너 몫.
