# 회원 도메인 — 시나리오 규정 (#10 결정 반영 초안)

> v2 · 2026-07-13 · 원저자: **민준(mypage 브랜치 `docs/scenario/user/`)** · 재구조화·#10 반영: 해찬
> ⚠ **오너 확정 대기** — #10 팀회의 결정([도메인경계_결정_2026-07-13](../도메인경계_결정_2026-07-13.md)) 반영 초안. 최종 확정 권한은 회원 도메인 오너(민준)에게 있음.

## 1. 도메인 정의
- **역할**: 마이(프로필·모임 레벨) + 정산 허브(보낼/받을 돈 요약·실행) + 소셜(팔로우) + 설정. 원저자 노트: "마이 = 소셜 프로필 + 정산 허브의 결합"(결합 타당성은 원저자 검증 항목).
- **원저자 모델 전제**: "카카오페이 기반 회원 간 직접 송금 + 송금 여부 직접 체크(포인트·에스크로 없음)" — 모임 스펙과 **수렴** ✅. 상태 3단(미송금→확인 대기→입금 확인)도 일치 ✅.
- **경계(원저자 명시)**: 상세 송금·정산 처리는 모임 도메인 화면으로 이동, 마이는 요약·진입점만. 내 모임 목록은 홈 소유(중복 배치 안 함) ✅.

## 2. 도메인 경계
- **(#10-1 확정)**: 정산 방식 주최자 선택 — 받을_돈 "환급 받기"·보낼_돈 "추가 정산"은 **사전 걷기 이벤트에서 유효**. 정산 실행은 모임 도메인 통합(결정 5).
- **용어**: "차수" 표기 잔존(일정·정산 내역) → 팀 정본 "정모"로 치환 필요(경미).
- **계좌 등록·인증**: 실계좌 인증 외부 API 전제 — 포인트용 KYC 폐기와는 별개 사안이나, 필요성 자체 재확인 권장(카카오페이 딥링크만 쓰면 계좌 등록이 필수인지 — 논의).
- **모임 레벨(신용등급)**: 점수 산정은 정산 도메인 이벤트가 입력(원저자 명시) — 모임 도메인과 이벤트 스키마 협의 필요.

## 3. 시나리오 (상세)

### 온보딩 — 계정 생성·첫 진입 인덱스

**개요**
첫 실행부터 홈 랜딩까지. 결정된 뼈대: **소셜 로그인 3종 → (가입 시) 전화번호 인증 → 닉네임 → 관심사 → 홈 빈 상태(추천 카드)**.

**상태 전이**
```
첫 실행 → 로그인(소셜 3종) ─ 기존 회원 ─▶ 홈
                        └ 신규 ─▶ 전화번호 인증 → 닉네임 → 관심사 → 홈(빈 상태·추천)
```

#### 소셜_로그인 — 카카오·구글·애플

**User Flow**
1. 첫 실행 → 로그인 화면 — 카카오(주) · 구글 · 애플(iOS 심사 요건)
2. 소셜 인증 → 기존 회원이면 홈 직행 · 신규면 전화번호_인증으로

**분기 / 엣지**
- 소셜 계정별 중복 가입(같은 사람이 카카오·구글 따로) — 전화번호 인증으로 동일인 병합 여부(논의)
- 로그인 실패·취소 → 로그인 화면 잔류
- 탈퇴 이력 계정의 재가입 처리(논의)

#### 전화번호_인증 — 신규 가입 1회

**User Flow**
1. 번호 입력 → 인증번호 SMS → 입력 → 확인
2. 성공 → 초기_설정으로 · 실패 → 재발송(횟수 제한, 논의)

**분기 / 엣지**
- 용도: 계정 신뢰(돈 앱) + 동일인 식별 — 카카오페이 송금 대상 확인에도 참고
- 번호 변경 시 재인증(계정_관리 연동)

#### 초기_설정 — 닉네임·관심사·랜딩

**User Flow**
1. 기본 닉네임 입력(전역 닉네임 — 모임별 닉네임은 가입폼에서 별도, #9)
2. 관심사 카테고리 선택 — 홈 추천 로직의 데이터 소스(예경 D6·콜드스타트와 연결)
3. 완료 → **홈 빈 상태 랜딩**(추천 모임/번개 카드 노출)

**분기 / 엣지**
- 관심사 스킵 허용 여부(논의 — 추천 품질과 트레이드오프)
- 프로필 사진·소개는 온보딩에서 받지 않음(마이 프로필_편집에서)

### 마이페이지_개요 — 허브 열람·분기 (leaf)

**개요**
마이 탭 진입 → 프로필·보낼 돈·받을 돈·일정·정산 내역을 한 화면에서 확인하고 하위 흐름으로 분기.

**행위자**
전원

**트리거 / 진입**
탭바 `마이`

**User Flow**
1. 프로필(아바타·이름·모임 레벨 배지·통계) 확인
2. 최근 정산 내역(개인 기록) → 정산_내역 · 내 모임 일정 → 내_일정
3. 톱니바퀴 → 설정
※ 보낼/받을 돈 요약은 **홈으로 이동(리뷰 결정 1)** — 마이에서 제거

**분기 / 엣지**
- **결합 질문 해소(#10-6 확정 — 하이브리드)**: 돈 **액션**(회비 송금·마감 todo)=홈 · 돈 **기록**(정산 내역·모임 레벨)=마이. 돈의 집을 홈으로 단일화하지 않고 액션/기록 축으로 분리. ⚠ 예경 P3 번복 연동 — 예경 confirm

### 모임_레벨 — 정산 신뢰 표시 인덱스

**개요**
정산 이력을 캐릭터·상태명·점수로 읽는 표시 전용 화면. 금융 신용등급 아님(원저자 명시). 점수 산정 입력은 정산 도메인 이벤트 — 스키마 협의 필요.

#### 레벨_확인 — 상태·점수 열람

**User Flow**
1. 마이 프로필의 레벨 배지 탭 → 상세
2. 캐릭터·상태명·점수·진행 막대·다음 단계 확인
3. 이력 없는 신규는 `첫 모임 · 이력 쌓는 중`(낮은 점수 ≠ 불이행 — 원저자 UI 계약)

**분기 / 엣지**
- 점수는 편집 불가(원저자 확정) · **공개 범위 = 공개(리뷰 결정 2)** — 상대 프로필에도 표시, 신규는 `이력 없음` 구분
- (논의·원저자) 분쟁·장애·예외의 감점·회복 기준 미정 — 확정 전 `확인 중` 상태 분리
- 모임 스펙과 접점: 미납·지연 이벤트가 입력 — 정산 도메인과 이벤트 스키마 협의(⚠경계)

#### 지연_회복_안내 — 정산 지연 상태

**User Flow**
1. 담당 정산 마감 경과 → `정산 확인 필요` 상태(흐린 캐릭터+시계 핀)
2. 지연 일수·회복 안내 확인 → 정산 완료 시 회복

**분기 / 엣지**
- 신규(이력 없음)와 지연을 문구·시각으로 구분(원저자 UI 계약 보존)
- 알림 설정의 정산 지연 알림과 연결 예정(원저자)

### 프로필_편집 — 프로필 수정 (leaf)

**행위자**
본인

**트리거 / 진입**
마이 프로필 [편집]

**User Flow**
1. 사진·닉네임·자기소개(한 줄)·연락처 수정
2. 저장 → 마이 복귀·즉시 반영

**분기 / 엣지**
- 실명/CI·모임 레벨은 편집 불가(원저자 확정)
- **실명 = 본인에게만 조회(리뷰 결정 3)** — 타인에겐 닉네임만. 정산 명단에서의 실명 표시 여부는 팀회의 후보(동명이인 혼동 vs 프라이버시)
- 프로필 사진 신고·모더레이션 연계(#10 신고 소속과 연결, 논의)

### 소셜_프로필 — 팔로우·게시글 인덱스

**개요**
마이 통계(게시글/팔로워/팔로잉)에서 목록·상대 프로필로 진입, 팔로우/언팔로우 수행.

#### 내_게시글_열람 — 내 글 모아보기

**User Flow**
1. 마이 `게시글 N` 탭 → 내 게시글 목록
2. 글 탭 → 게시글 상세(피드 도메인 F2)

**분기 / 엣지**
- 소스는 피드 도메인과 공유, 마이는 "내 것 필터" 조회만(원저자 확정 — 피드 스펙과 정합 ✅)

#### 팔로우_관리 — 팔로워·팔로잉 목록

**User Flow**
1. `팔로워 N`/`팔로잉 N` 탭 → 목록
2. 사용자 탭 → 상대_프로필_열람 · 팔로잉에서 언팔로우

**분기 / 엣지**
- **팔로워/팔로잉 전체 공개(리뷰 결정 4)** — 비공개 계정은 MVP 제외
- 차단/신고 사용자 관리 위치(누락 리뷰에서 다룸)
- 피드 소스(팔로우한 사람 글)의 그래프 원천 — 피드 스펙과 정합 ✅

#### 상대_프로필_열람 — 타인 프로필

**User Flow**
1. 목록/피드/모임 멤버에서 사용자 탭 → 상대 프로필
2. 팔로우/언팔로우 · 게시글 열람 · **모임 레벨 표시(리뷰 결정 2 — 공개)**

**분기 / 엣지**
- (논의·원저자) 팔로우↔DM 가능 조건(상호 팔로우?) — DM은 회원 소유
- 모임 멤버 관리에서의 프로필 연결(모임 스펙 멤버 관리 접점)

- 모임 스펙 정산 아카이브와 데이터 소스 공유 확인(⚠경계 — 같은 기록의 개인 단면)

### 알림_센터 — 통지 목록·딥링크 (leaf)

**개요**
전 도메인의 알림을 단일 목록으로 수신·열람하고 원인 화면으로 딥링크. **트리거 정의는 각 도메인 소유**(모임: 공통/알림 매핑 표 · 피드·홈 동일) — 이 문서는 목록 화면의 동작만 정의(중복 정의 금지).

**행위자**
본인

**트리거 / 진입**
홈 벨 아이콘(홈 도메인 진입점) · 푸시 탭

**User Flow**
1. 단일 목록(시간순) — 종류 배지로 구분(정산/모임/소셜), 별도 탭 없음
2. 유사 알림 그룹핑(같은 글 좋아요 N건 = 1줄)
3. 항목 탭 → 읽음 + 원인 화면 딥링크 · `모두 읽음` 버튼
4. 딥링크 대상 만료 → 폴백(정산 결과 → 지난 이벤트 → 모임 홈 순 — 홈/딥링크_폴백과 공유)

**분기 / 엣지**
- **보관 30일**(자동 정리) — 기록의 정본은 정산 아카이브·개인 내역이므로 알림은 사본
- 필수 알림(정산)은 설정에서 끌 수 없음(리뷰 결정 9와 연동)
- OS 푸시 권한 꺼짐 → 인앱 목록은 정상, 배너로 시스템 설정 유도(알림_설정 연동)

### 개인_채팅 — 1:1 대화 (leaf)

**개요**
1:1 개인 채팅. **시작 제한 없음**(누구나) — 안전장치는 **차단·신고**(#10-7 확정, 아래 시나리오). 인프라는 이벤트 채팅(모임 도메인)과 공유.

**행위자**
회원 누구나

**트리거 / 진입**
채팅 탭(목록) · 상대 프로필의 `채팅` · 모임 멤버/참여자 명단에서

**User Flow**
1. 채팅 목록 — 개인 대화 + 이벤트 채팅방 통합 표시(정렬: 최신 메시지순)
2. 상대 선택/프로필에서 시작 → 대화(텍스트·사진 — 범위 논의)
3. 나가기(목록에서 제거) · 차단·신고(차단·신고 시나리오)

**분기 / 엣지**
- 제한 없음의 안전장치: 차단 + 첫 메시지 수신 동의(메시지 요청함) 필요 여부(논의)
- 알림: 채팅 알림 카테고리(알림_설정의 소셜과 별도 여부, 논의)
- 팔로우와 무관(제한 없음 확정으로 팔로우 조건 논의 종결)

### 차단·신고 — 사용자 안전 (신설 · #10-7 확정)

**개요**
개인 채팅 "시작 제한 없음"의 선행 안전장치. **풀 모더레이션** 채택 — 차단(즉시 효과) + 사람 신고 접수 + 운영자 처리 큐를 MVP에 포함. 콘텐츠(게시글/댓글) 신고와 **통합 운영자 신고함**으로 처리(#10-4).

**행위자**
회원(차단·신고) · 플랫폼 운영자(처리)

**트리거 / 진입**
상대 프로필 · 개인/이벤트 채팅 · 게시글/댓글의 `차단`/`신고`

**User Flow**
1. **차단**: 상대 → `차단` → 즉시 효과(DM 차단·상호 노출 차단) · 설정의 차단 목록에서 해제
2. **신고**: 대상(사람/글/댓글) → `신고` → 사유 선택 → 접수 → 운영자 신고함
3. **운영자 처리**: 신고함 큐 → 경고·정지 등 조치(운영자 콘솔)

**분기 / 엣지**
- ⚠ 운영자 **역할·콘솔**은 신규 빌드 범위(운영 인력 전제) — 신고 소속(#10-4)과 공유
- 차단은 상호 은닉(양방향) · 신고는 대상 무통지
- 콘텐츠 신고 1차 처리는 모임 관리자(피드 F4), 에스컬레이션은 운영자(통합 신고함)

### 설정 — 알림·계정·계좌 인덱스

**개요**
마이 톱니바퀴로 진입하는 설정 허브. 원저자 4개 문서를 하위 행위로 통합.

#### 알림_설정 — 카테고리 토글

**User Flow**
1. 설정 → 알림 설정 → 카테고리(정산·송금 요청·미납/독촉·소셜) 상태 확인
2. 토글 변경 → 즉시 저장 (실패 시 이전 상태 복원+오류 안내 — 원저자)

**분기 / 엣지**
- **정산 필수 알림 고정(리뷰 결정 9)** — 송금 요청·입금 확인·반려는 토글 비활성 + "정산 알림은 끌 수 없어요" 사유 표시. 독촉·소셜만 선택
- OS 알림 권한 꺼짐 → 시스템 설정 연결(원저자)
- 알림 트리거 정의는 모임(공통/알림)·홈(알림_확인)과 3원 정의 — **매핑 표 통합 필요(논의)**

#### 계정_관리 — 연결 상태·로그아웃

**User Flow**
1. 계정 설정: 카카오 로그인·전화번호 인증·등록 계좌 상태 확인
2. 로그아웃 → 확인 → 세션 종료 → 온보딩 복귀

**분기 / 엣지**
- 계좌 항목 → 계좌_등록_인증으로 분기
- "전원 로그인 상태" 제품 전제하에 로그아웃 직후 상태는 온보딩만(모임 스펙 전제와 정합)

#### 회원_탈퇴 — 사전 검사·탈퇴

**User Flow**
1. 탈퇴 선택 → **사전 검사(리뷰 결정 10)**: **내가 주최한 미마감 정산만 차단**(이관 또는 마감 후 가능) — 참여자 미납·확인 대기는 차단하지 않음(기록 실명 유지, 교착 원천 제거)
2. 조건 충족 → 사유 입력 → 최종 확인 → 탈퇴 요청

**분기 / 엣지**
- 결정 10으로 원저자 조율 항목 해소 — 모임 탈퇴 규칙(미납 고지·명단 유지)과 일관
- 데이터 보관·파기(CI 등)는 인증 도메인 소관(원저자) · 탈퇴자의 모임 내 기록은 "실명 유지+탈퇴 표시"(모임 스펙 확정)와 정합 확인

#### 계좌_등록_인증 — 실계좌 인증

**User Flow (원저자안 보존)**
1. 등록 계좌·인증 상태 확인 → 은행·계좌번호 입력 → 인증 요청
2. 인증 코드/예금주 확인 → 성공 시 `인증됨` 반영 · 실패 시 사유 안내·재시도

**분기 / 엣지**
- **예금주 인증 유지(리뷰 결정 8)** — 폴백 수취 계좌의 오송금 방지가 돈 앱 신뢰의 기본
- 외부 API·재시도 횟수·변경 이력 보관 책임(원저자 검증 항목)

## 4. 검증 로그

**리뷰 결정 1(2026-07-12, 해찬 리뷰 — 팀 제안)**: **돈 현황의 집 = 홈**. 보낼/받을 돈 요약·진입은 홈으로 이동 제안(예경 P3 "회비 현황→마이" 번복 — #10 안건 추가 필요). 마이 = 프로필·소셜·정산 기록(개인 아카이브)·설정. 정산_허브 중 보낼_돈·받을_돈의 진입점은 홈, 정산_내역은 마이 유지. 근거: 제품 핵심(회비 투명성)이 2뎁스로 밀리는 것 방지 + 홈 todo와 마이 보낼 돈의 중복 제거.

**리뷰 결정 2(2026-07-12)**: 모임_레벨 **원안 유지**(점수·캐릭터 — 민준 설계 존중, 점수 구간·증감 정책은 정산 도메인이 확정 후 전달) + **공개 범위 = 공개**(상대 프로필에도 레벨 표시. 신규는 '이력 없음'으로 구분해 낙인 방지 — "제재보다 기록 노출" 원칙).

**리뷰 결정 3·4(2026-07-12)**: 실명 = **본인에게만 조회**(타인에겐 닉네임만 · 정산 명단의 실명 필요 여부는 팀회의 후보) · 팔로워/팔로잉 목록 = **전체 공개**(비공개 계정은 MVP 제외).

**리뷰 결정 5·6(2026-07-12)**: 보낼_돈·받을_돈 → **모임 도메인 정산으로 통합**(흐름 단일 정의, 회원엔 포인터만 — 3중 정의 해소) · 정산_내역 = **회원 유지, 영구 보관, CSV 내보내기 포함**(총무 인수인계·외부 장부용).

**리뷰 결정 7(2026-07-12)**: 일정 = **홈 단일화**(예경 스트립+인라인 필터안이 정본, 마이 주간 캘린더 제거 — 원저자 양측의 "중복 정리 필요" 메모 해소). 역할 정리: **홈=현황·행동(돈·일정·todo), 마이=정체성·기록·설정.**

**리뷰 결정 8~10(2026-07-12)**: 계좌 등록 = **예금주 인증 유지**(원안 존중 — 오송금 방지) · 정산 필수 알림(송금 요청·입금 확인·반려) = **끌 수 없음 고정 + 사유 표시** · 탈퇴 차단 = **내가 주최한 미마감 정산만**(이관·마감 후 가능), 참여자 미납·확인 대기는 차단 안 함(기록 실명 유지로 충분 — 교착 원천 제거).

**리뷰 결정 11(2026-07-12) — 온보딩 신설(공동 결정)**: 로그인 = 카카오·구글·애플 + 신규 가입 시 전화번호 인증 · 스텝 = 로그인→닉네임→관심사→홈 · 랜딩 = 홈 빈 상태(추천 카드, 예경 D6 연결).

**리뷰 결정 12(2026-07-12) — 알림_센터 신설(공동 결정)**: 소유=회원 확정 · 보관 30일 · 개별/전체 읽음 + 유사 그룹핑 · 단일 목록+종류 배지(탭 분리 없음) · 트리거 정의는 각 도메인 매핑 표 참조(3원 정의 해소).
