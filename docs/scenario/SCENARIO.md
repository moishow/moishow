## 1. 제품 전제

- **모바일 앱** — 모든 사용자는 로그인 상태(비로그인 플로우 없음). 딥링크 수신자가 앱 미설치면 스토어 유도.
- **돈은 앱을 거치지 않는다** — 포인트·에스크로·원장 없음(KYC 부담으로 폐기). 실제 이체는 카카오페이(외부), 앱은 **기록·확인·공개**로 신뢰를 만든다.
- **관통 원칙**: 기록은 지우지 않는다(취소·탈퇴·정정도 기록+표시) · 시스템 제재보다 기록 노출 · 결정에는 근거를 적는다.
- **용어 정본**: 커뮤니티 실체 = **모임**(구 동아리) · 이벤트 = **정모**(정기, 모임장·관리자 개설) / **번개**(즉흥, 부원 누구나 개설 · 내부/공개 타입). 폐기 어휘: 차수·에스크로·원장·포인트·잉여 환급.

## 2. 핵심 모델

### 정산 — 사후 정산 단일
**주최자가 먼저 지출 → 이벤트 후 실지출 기준 1/N(개인별 수정, 합계 일치 강제) 걷기.** 사전 걷기·잉여 환급·부족분 개념 없음.
- 납부 확인 2단계: 참여자 **'납부했어요' 셀프 신고** → 주최자 카카오페이 대조 → **확인 / 반려**(3상태: 미납·확인 대기·확인)
- 실지출 영수증 선택 첨부(`영수증 없음` 표시) · 납부 기한 + **기한 연장 신청**(연장 기한까지 마감 잠금, 완납 시 `완료·연장 N회` 기록) · 리마인드 1일 1회 · 노쇼 페널티 수입 = 실참여자 분담 차감
- 이의는 납부와 병행 가능, 반복 한도 없음 · 마감 후 오류는 **정정 기록 append**(원본 잠금)
- **강제 마감 허용**(미납은 기록으로 잔존) → 마감 시 **아카이브 공개**: 전 부원이 실지출·영수증·인별 금액(전부 공개)·정정 이력 열람

### 역할 — 모임장 / 관리자
- **모임장**(슈퍼유저): 모임 생성·삭제·승계·정보 수정 + 전권 · **관리자**: 정모 주최·정산·공지·가입 승인·강퇴 등 운영 실무
- 정산 책임은 역할이 아닌 **주최 행위에 귀속**(관리자 해제·부원 탈퇴에도 자기 정산은 마감까지 책임, 탈퇴 시 모임장 이관)
- 잠수 모임장: 관리자 요청 → 1주 간격 3회 알림 → 무응답 시 자동 승계

### 멤버십 — 재가입 3단계
가입 방식은 개설 시 선택(**즉시 가입 / 승인제**+폼빌더 — 기본 필수 항목: **모임별 닉네임**).
| 이력 | 재가입 |
|---|---|
| 일반 탈퇴 | 자유 |
| 강퇴 | 승인제 강제 |
| **블랙리스트**(강퇴 시 체크+사후 등록) | 신청 자체 차단 |
- 거절 재신청 쿨다운 7일(프리필) · 철회 즉시 재신청 · 초대 링크 경유자는 필수 항목만 작성+승인 생략

### 채팅 (DM 대체 · 2층 구조)
- **모임 상시방 = 모임 소유**: 모임당 1개 자동 생성·부원 자동 입장(단톡방 대체 완성)
- **이벤트 채팅 = 모임 소유**: 정모/번개 개설 시 자동 생성, 참여 시 자동 입장 → 정산 마감까지 활성 → 읽기 전용 아카이브
- **개인 채팅 = 회원 소유**: 1:1 제한 없음(차단이 안전장치 — ⚠차단·신고는 팀회의 잔여)

### 화면 배치 축 (리뷰 결정)
**홈 = 현황·행동**(오늘 챙길 일 · 보낼/받을 돈 요약 · 일정 — P3 번복 제안) / **마이 = 정체성·기록·설정**(프로필·모임 레벨·정산 내역·설정)
