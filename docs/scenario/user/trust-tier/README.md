# 모임 레벨

> 진입: 마이페이지 프로필의 `첫 모임 · 25점` 배지 ([[mypage-overview]])
> 화면: [`design/mypage/mypage/trust-tier.html`](../../../../design/mypage/mypage/trust-tier.html)
> 범위: **점수 정책이 아니라, 점수와 정산 상태를 사용자가 이해하는 UI 흐름**을 정의한다.

## 역할

모임 레벨은 금융 신용등급이 아니다. 사용자가 맡은 모임 정산의 결과를 캐릭터·상태명·점수로 읽고, 정산이 늦어진 경우에는 원인과 회복 필요성을 확인하는 마이페이지 하위 화면이다.

## User Flow

1. 마이페이지 프로필의 모임 레벨 배지를 누른다.
2. 현재 점수와 대응하는 캐릭터, 상태명, 다음 단계 정보를 확인한다.
3. 정산 이력이 없는 사용자는 `첫 모임 · 이력 쌓는 중`으로 확인한다.
4. 정산을 기한 내 완료해 기록이 쌓이면 상위 캐릭터와 다음 단계까지 남은 활동을 확인한다.
5. 정산 마감이 지난 경우 `정산 확인 필요` 상태에서 지연 일수와 회복 안내를 확인한다.

## 상태별 UI 계약

| 상태 | 예시 표시 | 핵심 표현 | 사용자가 알아야 할 것 |
|---|---|---|---|
| 신규 | 파랑 캐릭터 · `첫 모임` · `25점` | `이력 쌓는 중` | 낮은 점수는 불이행이 아니라 정산 이력이 아직 없다는 뜻이다. |
| 성장 | 코랄 캐릭터 · `약속 지킴이` · `72점` | `정산 6회 완료` | 누적 기록과 다음 캐릭터까지 남은 활동을 확인한다. |
| 정산 지연 | 흐린 캐릭터 + 시계 핀 · `정산 확인 필요` · `15점` | `정산 마감이 2일 지났어요` | 신규 상태와 구분되며, 정산 완료 뒤 회복할 수 있음을 안내한다. |

위 점수와 캐릭터는 현재 HTML 시안의 예시값이다. 실제 구간, 증감 폭, 공개 범위, 제재 기준은 정산/총무 도메인 정책에서 확정한다.

## 핵심 기능

- 프로필의 소형 캐릭터 배지에서 상세 화면으로 진입
- 캐릭터·상태명·점수·진행 막대를 함께 표시
- 신규 사용자와 정산 지연 사용자를 별도의 문구와 시각 상태로 구분
- `정산 확인 필요`에 시계 SVG 상태 핀과 회복 안내를 제공
- 상태 비교용 탭: `처음 시작` · `기록 쌓임` · `정산 지연`

## 데이터와 정책 경계

- 시작점 `25점`과 예시 점수 `72점`·`15점`은 UI 시안을 위한 값이다.
- 점수 변경은 시간이 흐르는 것만으로 발생하지 않는다. 담당 정산의 마감·완료·지연 같은 정산 도메인 이벤트가 입력이 된다.
- 분쟁, 시스템 장애, 정책상 예외의 감점 여부와 회복 기준은 아직 미정이다. 확정 전에는 UI에서 `확인 중` 상태로 분리해야 한다.
- 사용자는 점수를 직접 편집할 수 없으며, 프로필 편집 범위에도 포함하지 않는다.
