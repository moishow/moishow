# 받을 돈 — 걷기(입금 확인) + 환급 받기

> 진입: 마이페이지 "받을 돈" 섹션 ([[mypage-overview]])
> 세 종류: ① 내가 주최자로서 걷기 ② 총무가 나눠주는 환급 받기 ③ **참가 취소 환불 받기**

## User Flow A — 걷기 (내가 주최)
1. 마이 "받을 돈"의 걷기 행(예: `금요 번개 · 3/5명 송금 · 95,000원`) 탭 → **수금 현황**
2. 참여자별 송금 상태 체크리스트 확인 (송금함 / 미송금)
3. 각자 카카오페이로 나에게 보낸 것을 **내가 "입금 확인" 체크**
4. 전원 확인되면 걷기 완료 → 정산 진행

## User Flow B — 환급 받기
1. 총무가 차수 정산 완료(잉여 균등 환급) → **환급 예정** 행 노출 (`지급 대기`)
2. 총무가 카카오페이로 환급 송금 → 부원이 입금 확인
3. 상태 `지급 대기` → `입금 확인`, "최근 정산 내역"에 `+금액` 기록

## User Flow C — 참가 취소 환불 받기
1. 부원이 일정/차수 **참가 취소** → 환불 요청 생성
2. 부원의 "받을 돈"에 **`환불 대기`** 항목 노출 (주최자 확인 후 송금)
3. 주최자의 "보낼 돈"에 **`환불 요청`** 항목 노출 → **송금 상세(pay-detail) 재사용**으로 카카오페이 송금
4. 주최자가 "송금 완료" 체크 → 부원이 입금 확인 → `입금 확인`, 정산 내역에 `+금액` 기록

## 연결되는 페이지
- 마이페이지 (진입점)
- 수금 현황 (`#collect-detail`) — 참여자별 입금 확인 체크리스트
- 받을 돈 전체 (`#receivables`) — 걷기 · 환급 예정 · **환불 대기**
- 송금 상세 (`#pay-detail`) — 주최자가 환불 보낼 때 **재사용** ([[send-money]])
- 정산 내역 ([[settlement-history]])

## 핵심 기능 (필수)
- 주최자용 참여자별 송금/입금 체크리스트 (송금 여부 확인의 "받는 쪽")
- 환급 예정·완료 상태 노출
- **참가 취소 → 환불 대기 상태 노출** (환불 전용 페이지 없이 pay-detail 재사용)

## 비고 (검증 필요)
- **취소 진입점**: 일정 상세([[schedule]])의 "참석 취소"가 환불 요청을 만든다.
- 주최자가 환불을 미루면 강제 수단이 없음(P2P) → `환불 대기` 지속 + 독촉 알림만.
- **걷기 주체 범위**: 총무뿐 아니라 개인 벙을 연 부원도 걷을 수 있음 → 권한/역할 처리.
- 환급이 "총무가 직접 송금"이라 지연·누락 가능 → 미지급 추적·독촉 정책 필요 여부([[settings]] 알림).
- 부분 미송금 상태에서 걷기 마감/정산 강행 규칙.
