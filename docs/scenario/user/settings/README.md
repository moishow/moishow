# 설정 — 알림 · 계정

> 진입: 마이페이지 상단 **톱니바퀴** (마이 본문 메뉴가 아님) ([[mypage-overview]])

## User Flow
1. 마이 우상단 톱니바퀴 → **설정**
2. 분기:
   - **알림 설정**: 카테고리별 토글 (정산 · 송금 요청 · 미납/독촉 · 소셜(쇼·DM))
   - **계정 설정**: 앱 정보(버전/약관/개인정보처리방침), 로그아웃, 회원 탈퇴

## User Flow — 로그아웃 / 탈퇴
- 로그아웃 → 확인 → 온보딩(W-01) 복귀
- 회원 탈퇴 → 미송금·받을 돈·진행 중 일정 경고 → 사유 확인 → 탈퇴 요청

## 연결되는 페이지
- 마이페이지 (진입점 · 톱니바퀴)
- 설정 (`#settings`)
- 알림 설정 / 계정 설정
- 온보딩 (W-01, 로그아웃 복귀)

## 핵심 기능 (필수)
- 알림 카테고리별 on/off
- 로그아웃 · 회원 탈퇴 진입(민감 기능 하단 배치)

## 비고 (검증 필요)
- **끌 수 없는 필수 알림**: 송금 요청·입금 확인처럼 정산에 필수인 알림은 토글 비활성 필요 여부.
- **탈퇴 처리**: KYC/CI 데이터 폐기·보관 의무는 가입·인증 도메인 소관 → 마이는 "탈퇴 요청 UI"까지. 미송금/받을 돈 남은 상태의 탈퇴 허용 여부는 정산 도메인과 조율.
- 알림 카테고리 분류 기준 팀 합의 필요.
