# SCENARIO.md — 모이쇼 시나리오 개요 (리뷰용 · L0)

> 2026-07-13 · **리뷰·합의용 개요 레이어.** 제품 전제 · 도메인 지도 · 시나리오 인덱스 · 결정 현황만 담는다. 상세 User Flow·분기는 각 **도메인 스펙(L1)** 참조 — 한 사실은 한 곳에만 둔다(정합 유지 비용 최소화).
> **구조**: 개요(이 문서, `README.md`) · **도메인 스펙** [홈](domains/홈.md)·[회원](domains/회원.md)·[피드](domains/피드.md)·[모임](domains/모임.md) (`domains/`). 각 스펙이 **현재 확정본** — 결정은 결과·근거를 스펙에 인라인 반영(`#10-N 확정` 마커)하고, 별도 결정 로그·이전 버전 기록은 두지 않는다.

## 0. 제품 전제 (전 도메인 공통)

- **모바일 앱** — 모든 사용자는 로그인 상태(비로그인 플로우 없음). 딥링크 수신자가 앱 미설치면 스토어 유도.
- **돈은 앱을 거치지 않는다** — 포인트·에스크로·원장 없음(KYC 부담으로 폐기). 실제 이체는 카카오페이(외부), 앱은 **기록·확인·공개**로 신뢰를 만든다.
- **관통 원칙**: 기록은 지우지 않는다(취소·탈퇴·정정도 기록+표시 전환) · 시스템 제재보다 기록 노출 · 결정에는 근거를 적는다.
- **용어 정본**: 커뮤니티 실체=**모임**(구 동아리) · 이벤트=**정모**(정기)/**번개**(즉흥, 내부/공개) · **차수**=이벤트 하위 단위(1차/2차 — 차수별 시간·장소·정원, 차수 단위 신청 · 리뷰 2026-07-14 재도입). 폐기 어휘: 에스크로·원장·포인트. ※ 환급·추가정산은 **사전 걷기** 이벤트에서 유효(#10-1 주최자 선택).
- **역할**: **모임장**(슈퍼유저) / **관리자**(정모 주최·정산·공지·가입 승인·강퇴 등 실무) / 부원 / 비부원 참여자. 정산 책임은 **주최 행위에 귀속**.
- **화면 배치 축**: 홈=현황·행동(오늘 챙길 일·보낼/받을 돈·일정) / 마이=정체성·기록·설정.
- **채팅 2층 구조**: 모임 상시방(모임당 1) + 이벤트방(정모/번개당 1) = 모임 도메인 · 개인 1:1 = 회원 도메인.
