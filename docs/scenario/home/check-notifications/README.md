# home / check-notifications — 알림 확인

## 시나리오 개요
홈 우상단 **벨 아이콘(빨간 점 = 미확인)** 에서 알림 센터로 진입해, 가입 승인·송금 확인/미송금·정산·환급 등 각종 통지를 확인하고 **원인 화면으로 딥링크**한다. 알림은 회비 행동을 유발하는 주요 재진입 경로다.

## 페르소나
멤버 · 총무 (공통)

## 트리거
- 홈 벨 아이콘 탭 (`bell__dot` 점등 = 미확인 존재)
- 푸시 알림 탭(앱 외부) → 알림 센터 또는 원인 화면 직행

## User Flow
1. 홈 → 🔔 벨 탭 → **W-26 알림**
2. 알림 목록 스캔(가입 승인, 송금 확인 완료/미송금 반려, 정산 완료, 환급 수령, 미납 독촉 등)
3. 항목 탭 → 원인 화면으로 딥링크
4. 처리 후 뒤로 → 알림 읽음 처리 → 벨 점 소거

## 연결 페이지
| 알림 종류 | 딥링크 목적지 | 도메인 |
|---|---|---|
| 가입 승인/반려 | W-07/모임 홈 | crew |
| 송금 확인 완료 / 미송금 반려(재송금) | W-11 / 회비 송금(GAP) | crew |
| (총무) 송금 확인 요청 N건 | W-TR | crew |
| 정산 완료 | W-23 | crew |
| 추가정산 요청 | W-22 | crew |
| 환급 수령확인 요청 | 회비 현황(GAP) | crew·user |
| 미납 독촉 | W-25 | user |
| DM 수신 | W-30 | user |
| 신고/모더 결과 | W-33 피드 상세 | crew |

## 분기 / 엣지
- **미확인 없음**: 벨 점 없음 → 알림 센터는 열리되 빈/과거 목록
- **딥링크 대상 만료**: 마감 지난 정모 알림 탭 → 정산 결과(W-23)로 폴백
- **알림 설정**: 종류별 on/off — *(논의: 설정 화면은 user 도메인. 홈에서 직접 진입 안 함)*

## GAP
- 알림 **설정(구독 관리)** 화면 — 현재 레지스트리에 없음. user 도메인에서 다룰지 결정.

## 검증 메모 (Step 2)
- 알림은 홈의 벨에서만 진입하는가, 마이(W-25)에도 알림 탭이 있는가 → 중복 진입점 정리.
- W-26이 "목록"만인지, 항목별 딥링크 규칙이 정의됐는지 확인(딥링크 매핑 표를 별도 문서화 권장).
