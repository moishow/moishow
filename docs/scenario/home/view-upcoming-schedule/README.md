# home / view-upcoming-schedule — 다가오는 일정 확인

## 시나리오 개요
홈의 **캘린더 + 다가오는 일정** 블록에서 내가 속한 모임의 정기모임·번개 일정을 확인하고, 특정 일정을 탭해 해당 정모 상세로 진입한다. "언제 무엇을 해야 하는가"의 시간 축 뷰.

## 페르소나
멤버 · 총무 (공통)

## 트리거
- 홈 캘린더 스트립의 날짜 탭(점 표시 = 일정 있음) — **좌우 스크롤로 여러 주 탐색 가능**
- "다가오는 일정" 이벤트 행 (예: "봄 시즌 킥오프·정기모임", "6월 마지막 풋살 · 번개")

## User Flow
1. 홈 → 캘린더 스트립을 좌우로 스크롤하며 점 찍힌 날짜 탐색(주 단위 페이징/캐러셀)
2. 날짜 탭 → 하단 "다가오는 일정" 리스트가 **해당 날짜 기준으로 인라인 필터링**
3. 이벤트 행 탭 → **W-11 정모 상세**
4. 상세에서 달성 게이지·만남시각·장소 확인 → 필요 시 회비 송금(check-todos와 접점)

## 연결 페이지
| 진입점 | 화면 | 도메인 | 비고 |
|---|---|---|---|
| 이벤트 행 | W-11 정모 상세 | crew | 정기모임·번개 공통 |
| 날짜 스트립 탭 | 홈 내 인라인 필터(페이지 이동 없음) | home | 별도 화면 불필요로 결론 |

## 분기 / 엣지
- **번개(즉흥) 일정**: badge "번개" — 정기모임과 진입 화면은 같으나(W-11) 생성 경로가 다름(crew 도메인)
- **일정 없는 날**: 점 없음 → 탭 시 "일정 없음" 빈 상태로 리스트 대체(무반응 아님)
- **만남 이후 일정**: 만남시각 지난 정모는 정산(OCR·환급) 단계로 전환 → check-todos·crew와 연결
- **월 경계 스크롤**: 스트립이 다음 달로 넘어가면 상단 월 텍스트(`2026년 7월` 등)가 자동 갱신 — 별도 "더보기" 버튼 제거

## GAP — 해소됨 (Step 2 결론)
~~전체 일정/월 캘린더 화면~~ → **불필요**. 스트립을 좌우 스크롤(멀티 위크 캐러셀)로 확장하고, 날짜 탭 시 하단 이벤트 리스트를 인라인 필터링하는 것으로 니즈를 충족. 별도 페이지·네비게이션 없이 홈 화면 안에서 완결.
- 근거: (1) 기존 구조(스트립+리스트)가 이미 같은 정보 구조라 화면 분리 불필요 (2) 모이쇼 핵심 가치(회비 투명성)와 무관한 스코프 확장 지양 (3) 모임별 상세 일정은 모임 홈(crew 도메인, 별도 GAP)에서 다룸 — 홈은 전체 모임 통합 미리보기만 담당

## 검증 메모 (Step 2)
- 일정의 **소스**: 정모(round)의 만남시각만인가, 아니면 별도 이벤트 개념이 있는가 — 데이터 모델 확인.
- ✅ [확정] 전체 캘린더 화면 스코프 제외. 스트립 스크롤 확장 + 인라인 필터로 대체.
