## 시나리오 개요
W-05는 멤버·총무 **공용 화면**이지만, 총무는 모임 운영을 위한 추가 진입점을 본다. 새 정산 모델에서 총무의 핵심 업무는 **멤버 송금 확인(완료/미송금 처리)** 과 **정모 종료·OCR·정산**이다. 홈에서 총무 전용 "오늘 챙길 일"과 내 모임 카드를 통해 **총무 대시보드(W-TR)** 로 핸드오프한다. 역할(RBAC)에 따라 홈이 다르게 큐레이션되는 지점.

## 페르소나
총무 (Leader)

## 트리거
- 홈 "오늘 챙길 일"의 총무 전용 카드
  - "송금 확인 N건" (멤버가 [송금완료] → 확인대기 상태)
  - "미송금 추적" (마감 지난 미납 멤버)
  - "정모 종료·정산 필요" (만남 종료 후 OCR)
- 내 모임 카드(예: "디자인모임 모디 · 정산 진행") 탭 → 모임 홈 → 관리

## User Flow
1. 총무 홈 진입 → 총무 전용 todo 확인
2. "송금 확인" 탭 → **W-TR 총무 대시보드** → 확인대기 목록에서 실제 입금 대조 → **완료 처리** / **미송금 처리(반려)**
3. 미송금 처리 → 멤버에게 "재송금 필요" 알림(W-26) 발송 → 멤버 재송금(check-todos 2-A)
4. 만남 종료 → **W-19 정모 종료 OCR**(영수증 실지출 확정) → 자동 정산
5. 잉여 → 멤버별 **역방향 송금 딥링크**로 환급 실행 → 멤버 수령확인(check-todos 2-C)
6. 적자 → **W-21 추가정산 요청** → 멤버 추가 송금(W-22)

## 연결 페이지
| 진입점 | 화면 | 이후 | 도메인 |
|---|---|---|---|
| 총무 todo(송금 확인) | W-TR 총무 대시보드 | 완료/미송금 처리 → W-26 알림 | crew |
| 총무 todo(정산) | W-19 정모 종료 OCR | W-21→W-22→W-23 | crew |

## 분기 / 엣지
- **권한(RBAC) 분기**: 같은 W-05라도 총무면 총무 todo·`관리` 노출, 멤버면 미노출
- **다중 모임 총무**: 여러 모임 총무일 때 todo가 모임별로 섞임 → 모임 컨텍스트 표기 필요
- **미송금 반려 반복**: 멤버가 계속 미송금이면 신용등급·프로필 표시로 압박(강제력 없음, S8)
- **환급 역송금 미수령**: 총무가 역송금했으나 멤버 수령확인 지연 → 알림 재발송

## GAP (신규 페이지 후보)
- **총무 송금 확인 화면(완료/미송금 처리)**: 신규 — check-todos와 공유하는 핵심 화면
- **멤버용 모임 홈 vs 총무 관리 뷰**의 관계 (crew 도메인 핵심 GAP)

## 진입 동선 (D2·확정)
- 총무 전용 항목은 **별도 메뉴가 아니라 "오늘 챙길 일" todo의 일부**로 노출(RBAC 분기)
- todo 탭 → **모임 홈 경유** → `관리`(W-TR) 진입. (홈 todo에서 W-TR 직행 아님 — 모임 컨텍스트를 먼저 잡음)
- 근거: 다중 모임 총무일 때 모임 컨텍스트 없이 W-TR 직행하면 어느 모임인지 모호

## 검증 메모 (Step 2)
- ✅ [확정] 10분 동의창·락·총무 출금(구 W-16/17/18) 제거. 총무 핵심 업무는 **송금 확인**으로 대체.
- ✅ [확정 D2] 총무 todo = 오늘 챙길 일의 일부(별도 메뉴 X), W-TR 진입은 **모임 홈 경유**.
- W-05를 멤버/총무 **공용 단일 화면 유지 + todo/진입점만 RBAC 분기**.
- 총무 전용 "오늘 챙길 일" 항목(송금 확인·미송금 추적·정산)을 멤버용(check-todos)과 분리 명세.
