# home / view-dashboard — 홈 대시보드 개관

## 시나리오 개요
사용자가 앱에 진입(온보딩 직후 또는 재진입)하면 홈에서 **내 상태를 한 화면으로 파악**한다. 오늘 챙길 일 건수, 다가오는 일정, 내 모임 현황을 스캔하고 다음 행동을 결정한다. 홈은 이후 모든 시나리오의 분기 기점이 된다.
> 회비 현황 요약 카드는 홈에서 제거(P3) — 긴급도는 인사 "오늘 챙길 일 N건" + 오늘 챙길 일 블록으로 전달하고, 상세 회비 현황은 마이(user 도메인)로 이관.

## 페르소나
멤버 · 총무 (공통)

## 트리거
- 온보딩 완료(S1) 후 자동 진입 → **홈 진입 수신점**
- 하단 탭바 `홈` 재선택
- 타 화면에서 뒤로가기 누적 → 홈 복귀

## User Flow
1. 앱 진입 → W-05 홈 로드
2. 상단 인사 + "오늘 챙길 일 N건" 요약 확인
3. 오늘 챙길 일 / 다가오는 일정 / 내 모임 / 추천 순으로 세로 스캔
4. 처리할 항목을 골라 해당 시나리오로 분기

## 연결 페이지
| 방향 | 화면 | 도메인 | 비고 |
|---|---|---|---|
| 진입 수신 | W-04 온보딩 완료 → **W-05** | user→home | S1 종점 |
| 분기 | 각 홈 시나리오(2~8) | home | 아래 각 README 참조 |

## 분기 / 엣지
- **신규 사용자(데이터 없음)**: 오늘 챙길 일 0건 · 내 모임 0개 → 빈 상태(empty state)에서 추천/탐색으로 유도 → `explore-recommendations`
- **미인증(KYC 미완)** 상태로 홈 진입 가능한가 → *(논의: S1은 KYC를 온보딩에 강제. 홈에서 KYC 재요구 지점은 없음)*

## GAP (신규 페이지 후보)
- 신규 사용자용 **홈 빈 상태** 디자인 (오늘 챙길 일/내 모임 0건) — W-05는 데이터 채워진 상태만 존재

## 검증 메모 (Step 2)
- 홈은 "조회" 시나리오이므로 자체 종점이 없다 — 모든 실질 행동은 2~8로 위임. 이 분리가 맞는지 확인.
- 빈 상태를 별도 시나리오로 뺄지, view-dashboard의 엣지로 둘지 결정 필요.
