# home 도메인 — 시나리오 인덱스

> 기점 화면: `design/system/screens/W-05.html` (홈)
> 도메인 정의: **진입·대시보드·라우팅 허브**. 홈이 트리거이자 주 서피스인 흐름을 담고, 깊은 작업(회비 송금·가입·정산 실행)은 각 도메인(crew/user)으로 **핸드오프**한다. 각 시나리오 README의 "연결 페이지"가 핸드오프 지점을 명시한다.

> ⚠️ **용어 정본**: 모임(구 동아리) · 정모(구 차수) · 피드(구 쇼).
> 관계: **모임에 멤버가 속한다 → 모임은 정모를 연다 → 번개는 모임에 속할 수도, 아닐 수도 있다.**

> ⚠️ **정산 모델 정본(간소화)**: 앱은 결제/에스크로를 쓰지 않는다. **송금 딥링크만 제공** →
> 멤버가 실제 계좌이체 후 **[송금완료]** 클릭 → **확인대기** → 총무가 실제 입금 대조 후 **완료/미송금 처리**.
> 잉여 환급·적자 추가정산도 동일한 송금+확인 방식. 영수증 OCR(W-19)로 실지출 확정은 유지.

## 도메인 분류(전체 서비스)

소유 주체 + 데이터 소유권(bounded context) 기준.

| 도메인 | 소유 주체 | 다루는 것 | 대표 화면 |
|---|---|---|---|
| **home** | 서비스(허브) | 진입·대시보드·오늘 챙길 일·일정·추천·라우팅 | W-05 |
| **user** | 개인(멤버 나) | 온보딩(로그인·KYC), 마이·회비 현황·신용등급, 알림, DM·신고·CS | W-01~04, W-25, W-26, W-29~32 |
| **crew** | 모임(집단) | 탐색·가입·개설·모임홈 + 정모(회비 송금·확인·정산·환급) + 피드 | W-06~23, W-27·W-28·W-33, W-TR |
| **admin** | 운영자 | 심사·감사·모더·CS·리포트 | W-40~47 |

> 홈에서 파생되는 플로우는 **home·user·crew** 3개만 건드린다(admin은 홈에서 진입 불가).
> - 커뮤니티(구 social)는 **피드=crew(모임 글), DM·신고·CS=user** 로 흡수.
> - 포인트 지갑 개념 제거 → 개인 자금은 앱이 보관하지 않으며, user 도메인은 **회비 현황·송금 상태**만 다룬다.
