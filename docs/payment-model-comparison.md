# 결제·정산 모델 비교노트 — 앱 포인트 에스크로 vs 라이선스 PG 경유

## 0. 왜 이 노트가 필요한가

모이쇼의 **자금 흐름 설계에 서로 다른 두 방향**이 존재한다.

- **A안 (기존 정본)**: 앱 포인트 + 기간 제한형 에스크로 + append-only 원장으로 자금을 **플랫폼이 직접 보관·정산**하되, 규제는 "선불업 등록 **유예** 조건"을 지켜 회피한다. (`problem.md` §5~6, 현재 와이어플로우 W-12~24)
- **B안 (minjoon 리뷰 2026-07-01)**: 자금을 플랫폼이 직접 보관하지 않고 **라이선스 PG(토스페이먼츠)의 정산 인프라를 경유**하고, CODEF 1원 계좌인증 + 포트원 PASS 본인인증으로 KYC/AML을 강화해 **전자금융거래법 위반을 원천 차단**한다.

두 안은 **부분적으로 상충**한다(특히 자금 보관 주체·수수료·핵심 UX). 지금 정렬하지 않으면 백엔드 구현이 어느 쪽 전제로 갈지 갈린다.

## 2. 상충 지점 (충돌하는 부분)

1. **자금 보관 주체 — 가장 큰 분기.**
   A안은 "플랫폼이 포인트를 보관·정산"이 핵심이고, B안은 바로 그 구조가 전자금융거래법(선불전자지급수단 발행/PG업) 위반이라 **PG 경유로 자금을 플랫폼 밖에 둔다**. 둘은 근본 전제가 반대다.

2. **"수수료 0" 원칙 vs PG 수수료.**
   A안의 선불업 유예 조건 중 하나가 **수수료 0원**이다(`problem.md` §6). B안의 PG 경유는 통상 결제/정산 수수료가 발생하므로, 유예 조건과 충돌하거나 수익모델을 다시 설계해야 한다.

3. **핵심 신뢰 UX의 성립 여부.**
   모이쇼의 차별점인 **10분 동의창·에스크로 5단계·노쇼 페널티·원장 검증**은 "플랫폼이 자금을 붙잡고 상태머신으로 통제"하는 데서 나온다. PG 정산 모델에서 이 통제권을 어떻게 확보할지 리뷰는 답하지 않는다.

4. **충전 채널 불일치.**
   현재 화면·와이어플로우는 **카카오페이 충전**(W-13)을 전제하는데, 리뷰는 **토스페이먼츠**를 든다. PG 벤더 선택이 미정렬.

## 3. 공통점 (양립 가능·둘 다 채택 가치)

- **강한 KYC/AML.** 리뷰의 PASS(실명+CI) + CODEF 1원 계좌인증 + 예금주명 교차검증은 A안을 택하더라도 **그대로 채택할 가치가 큰 개선**이다(자금세탁·명의도용 방어, 1인 1계정). A안의 W-03 KYC를 이 수준으로 구체화하는 것은 안전하게 합칠 수 있는 부분이다.
- **서버 측 무결성 검증.** 결제 금액 위변조 방지(서버 재검증), 멱등키, 웹훅 재조회는 어느 안이든 필수.

## 4. 결정이 필요한 열린 질문 (Open Questions)

1. **자금을 플랫폼 포인트 에스크로에 보관할 것인가, PG 정산을 경유할 것인가?** (핵심 분기 — 나머지가 여기에 종속)
2. 에스크로 5단계·10분 동의창·노쇼 페널티 같은 **구조적 신뢰 UX를 B안(PG)에서 어떻게 재현**하는가? (재현 불가면 제품 차별점이 약해짐)
3. **"수수료 0" 원칙을 유지하는가?** 유지 못 하면 유예 전략(A안)과 수익모델을 어떻게 조정하는가?
4. KYC를 **리뷰 수준(PASS+CI+1원 계좌인증 교차검증)으로 강화**할 것인가? (어느 안이든 권장)
5. 충전/결제 벤더를 **카카오페이 vs 토스페이먼츠** 중 무엇으로 정렬하는가?

## 5. 방향 제안 (중립 — 최종 결정은 팀)

- **즉시 합의 가능한 것**: KYC 강화(PASS+CI+CODEF 1원 계좌인증 교차검증)와 서버측 결제 무결성 검증은 **안 선택과 무관하게 지금 확정**해도 좋다.
- **하이브리드 검토**: PG(토스)를 **충전·현금화 레일**로만 쓰고, 앱 내 **포인트/에스크로 상태머신(10분 동의·노쇼·정산)은 유지**하는 절충이 가능한지 검토. 단 이 경우도 "플랫폼이 포인트를 보관"하는 선불업 이슈가 남으므로 **금융 규제 법률 자문이 선행**되어야 한다.
- **문서 정본 정리**: 결정이 나면 `problem.md` §5~6과 화면(W-12~24)을 선택안에 맞춰 갱신하고, 이 비교노트에 "결정: X안(YYYY-MM-DD)"을 기록한다.
