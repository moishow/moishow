# 모이쇼 고객여정지도 (Customer Journey Map)

> 버전 v0.1 · 2026-07-01
> 근거: `docs/problem.md`(문제 정의·6단계 위험구간·핵심문제 5개) · `docs/wireframe.html`(시나리오 S1~S8·화면 레지스트리 W-xx)
> 목적: "개인 계좌 + 단톡방 캡처"로 운영되던 회비/펀딩을, 부원·총무·운영자 세 페르소나의 눈으로 자금 6단계를 따라가며 **어디서 신뢰가 끊기는지(As-Is)**와 **모이쇼가 그것을 구조적으로 어떻게 검증 가능하게 바꾸는지(To-Be)**를 한 장에 잇는다.

## 0. 이 지도를 읽는 법

### 페르소나 3종

| 페르소나 | 한 줄 니즈 | 근본 불안 |
|---|---|---|
| **부원 (Member)** | 내가 낸 회비가 어디에 어떻게 쓰였는지 스스로 확인하고, 남으면 총무 선의에 기대지 않고 공평하게 자동 환급받는다 | 내 돈이 총무 개인계좌에 무기한 묶여 떼이거나(먹튀) 불투명하게 쓰일까 — 정작 이의를 제기할 로그는 총무에게만 있고 나에겐 없다 |
| **총무 (Leader)** | 회비를 빨리 걷고 간편히 쓰고, 정산을 자동으로 끝내 '먹튀 의심' 없이 책임에서 벗어난다 | 남의 돈이 내 개인계좌에 머무는 동안 '떼먹었다'는 의심을 받고, 밤새 수기 정산해도 계산 실수 하나가 곧 먹튀 누명이 된다 |
| **운영자 (모이쇼 플랫폼/어드민)** | 모든 자금 이동을 append-only 원장으로 남겨, 규제 유예 조건을 지키면서 모든 분쟁을 '진술'이 아니라 '조회'로 끝낸다 | 플랫폼이 '송금 대행'처럼 보이면 선불업 등록 의무가 발생하고, 총무 먹튀·정산 지연이 곧 플랫폼 신뢰도로 직결되며, 원장이 없으면 모든 분쟁이 진술 싸움이 되어 운영자가 근거 없이 책임을 뒤집어쓴다 |

### 자금 라이프사이클 6단계 = 여정의 척추 (problem.md §4)

```
① 모집  →  ② 납부  →  ③ 보관  →  ④ 지출  →  ⑤ 정산  →  ⑥ 환급
```

여섯 단계 전 과정의 **공통 근본 원인**은 하나다 — "제3자가 검증 가능한 기록이 하나도 남지 않는다"(개인계좌 + 기억 + 단톡방 캡처). 모이쇼는 이 척추를 따라 **기간 제한형 에스크로**와 **append-only 원장**으로 각 단계를 검증 가능하게 만든다.

> 주의: 운영자(어드민) 여정의 ①②③(심사·자금흐름 감독·대사)은 **어드민 기능 단계**이며, 위 자금 6단계의 ①모집/②납부/③보관과는 다른 축이다. 혼동하지 말 것.

### 감정 점수 범례 (-5 ~ +5)

| 점수 | 의미 |
|---|---|
| +5 ~ +3 | 완전한 신뢰·만족 |
| +2 ~ +1 | 옅은 기대·편안함 |
| 0 | 중립 |
| -1 ~ -2 | 조심스러움·긴장 |
| -3 ~ -4 | 불안·경계·부담 |
| -5 | 극심한 불신·고통·공포 |

### As-Is / To-Be

- **As-Is**: 모이쇼 도입 전 — 총무 개인계좌로 걷고, 단톡방으로 확인·공지하고, 총무 수기로 정산·개별 송금하는 현재 방식.
- **To-Be**: 모이쇼 도입 후 — 앱 포인트 지갑·에스크로 잠금·append-only 원장·자동 정산으로 자금 흐름이 구조적으로 검증되는 방식.

## 2. 감정 곡선 (As-Is, 개인계좌+단톡방)

아래는 모이쇼 **도입 전** 부원·총무가 자금 6단계를 지날 때의 감정 점수다. 두 페르소나 모두 단계가 진행될수록 불신·부담이 깊어지며, 특히 부원은 ⑥환급(-5), 총무는 ⑤정산(-5)에서 최저점을 찍는다.

> 차트에는 범례가 표시되지 않는다. **첫 번째 라인 = 부원, 두 번째 라인 = 총무**이며, 정확한 값은 아래 표로 확인한다.

```mermaid
xychart-beta
    title "As-Is 감정 곡선 — 부원 vs 총무 (개인계좌+단톡방)"
    x-axis [모집, 납부, 보관, 지출, 정산, 환급]
    y-axis "감정 점수" -5 --> 5
    line [-1, -2, -3, -3, -4, -5]
    line [0, -2, -3, -2, -5, -3]
```

| 페르소나 | ① 모집 | ② 납부 | ③ 보관 | ④ 지출 | ⑤ 정산 | ⑥ 환급 |
|---|---|---|---|---|---|---|
| **부원** | -1 | -2 | -3 | -3 | -4 | **-5** |
| **총무** | 0 | -2 | -3 | -2 | **-5** | -3 |

- **부원**: 예산 깜깜이 입금(모집)에서 시작해 개인계좌 체류 불안(보관), 총무 임의 지출(지출)을 거쳐, "환급해준다더니 감감무소식"인 ⑥환급에서 최저점.
- **총무**: 시작은 "내 계좌 쓰면 편한데"(모집, 0)이지만 대사 노가다(납부)·남의 돈 떠안기(보관)를 지나, 밤새 수기 계산이 곧 먹튀 누명이 되는 ⑤정산에서 최저점.

## 4. 진실의 순간 (Moments of Truth)

여정 전체의 make-or-break 지점을 종합한다. 앞의 5개는 서비스 채택·리텐션을 가르는 자금 라이프사이클 상의 순간(spine)이고, 뒤의 3개는 운영자 여정에서 플랫폼 존립을 가르는 순간이다.

| 순간 | 왜 make-or-break | 화면 |
|---|---|---|
| **첫 예치 — 개인계좌 대신 에스크로에 처음 돈을 맡김 (OPEN)** | 부원이 As-Is의 총무 개인계좌 대신 앱 에스크로에 처음 돈을 넣는 진입점. '내 돈이 잠기고 만남 전엔 즉시 환불된다'는 안전감을 이 순간 체감하지 못하면 서비스 자체를 채택하지 않는다.<br>총무 관점: '내 통장에 남의 돈'이 '내 통장에 아예 없음(에스크로 잠금)'으로 바뀌어, 아무것도 안 해도 따라붙던 의심 자체가 사라진다. | W-14 |
| **만남 전 취소 → 즉시 환불** | As-Is 최대 불신 지점(환불 지연·먹튀)이 실제로 검증되는 순간. 즉시 환불이 작동하면 '언제든 뺄 수 있다'는 신뢰가 굳고, 지연되면 개인계좌와 다를 바 없다는 낙인이 찍힌다. | W-15 |
| **만남시각 락 후 10분 동의창 — 총무 출금 게이트** | 총무의 임의 출금을 구조적으로 막는 유일한 관문. 부원 동의(≥1)가 총무 출금의 전제이고 미응답은 노쇼 페널티로 이어진다 — 자금이 방출되느냐 VOIDED 전원 환불이냐가 갈리는 분기점.<br>총무 관점: 동의 게이트를 통과해야만 출금되고 그 사실이 원장에 남아, 출금 행위 자체가 총무의 결백 증거가 된다. | W-16<br>(→W-17→W-18) |
| **차수 종료 영수증 OCR → 자동 정산 완료 (DONE, 잔액 0)** | As-Is에서 신뢰가 가장 자주 깨지는 정산을 시스템이 대신 확정하는 순간. 실지출이 원장에 확정되고 잔액 0으로 닫히면 '계산 실수 vs 먹튀' 논쟁의 근거 자체가 사라져 총무가 의심에서 벗어나고, 부원은 '정직하게 쓰였고 남은 건 공평하게 돌아온다'를 검증한다. | W-23<br>(W-19) |
| **환급 수령 · 수수료 0원 현금화 + 원장 검증** | 부원이 낸 돈이 실제로 공정하게 돌아오는지 확인하는 최종 순간. 환급이 정확하고 원장으로 스스로 검증되면 재예치(리텐션)로 이어지고, 누락되면 영구 이탈한다. | W-24<br>(W-25) |
| **[운영자] 심사 통과 = 에스크로 접근 권한 부여** | 나쁜 총무를 입구에서 막느냐가 이후 모든 자금 사고를 예방하는 첫 관문 — 여기서 통과시킨 신원이 곧 에스크로 접근 권한이 된다. | W-41 |
| **[운영자] 원장으로 자금 흐름 제3자 검증** | 플랫폼의 존립이 걸린 순간 — 원장으로 자금 흐름을 제3자 검증할 수 있어야 규제 유예도, '먹튀 없는 플랫폼'이라는 신뢰도 함께 성립한다. | W-42 |
| **[운영자] 분쟁을 진술이 아닌 증거로 종결** | 분쟁을 진술이 아닌 증거로 끝낼 수 있는지가 플랫폼 신뢰도의 실전 시험대 — 원장이 여기서 실제로 '심판' 역할을 한다. | W-45 |

## 6. 기회 영역 & 다음 액션

여정 분석에서 도출되는 개선·검증 포인트다. 각 항목은 특정 화면/단계에 연결된다.

- **KYC 마찰 vs 전환율 (⓪ 온보딩 · W-03)**: KYC가 부원의 '무방비한 편안함'을 신뢰로 바꾸는 유일한 장치지만, 동시에 온보딩 이탈의 최대 후보다. 본인인증 단계의 완료율·이탈 지점을 계측하고, 첫 진입에서 KYC를 뒤로 미룰지(예: 첫 예치 직전) A/B로 검증할 것.
- **첫 예치 = 채택 분기점 (② 납부 → ③ 보관 · W-14)**: 진실의 순간 1번. '에스크로에 잠겼다'를 눈으로 확인시키는 순간의 카피·시각 표현(총무 출금 잠김 상태 명시)이 채택률을 좌우한다. W-14의 잠금 상태 가시성을 최우선 디자인 검증 대상으로.
- **10분 동의창 응답률·노쇼율 UX (④ 지출 · W-16)**: 동의자 0명이면 VOIDED 전원 환불로 차수가 무산되므로, 10분 창 도달률·응답률이 서비스 성패의 급소다. 노쇼율을 주최자가 설정하는 UX(기본값·상한·부원 사전 고지)를 정교화하고, 푸시 도달 실패 시 폴백(리마인드·창 연장 정책)을 검토할 것.
- **미납 강제력 부재의 실효성 검증 (⑤ 정산/엣지 · W-22→W-25→W-26)**: 규제·인권상 강제출금이 없어 미납 회수는 프로필 표시+지속 푸시에만 의존한다. 이 정책적 압박의 실제 회수율을 측정하고, 적자 추가정산이 반복 미납으로 고착될 때의 총무 구제책(예: 손실 분담 규칙 명문화)을 설계할 것.
- **충전 지연 관측성 (엣지 · W-13→W-43)**: 카카오페이 충전 지연/DLQ/멱등이 부원에게는 '예치 실패'로 체감된다. 재시도 상태를 부원 화면(W-13)에 실시간 노출하고, 어드민 대사(W-43)의 미해결 큐 SLA·알림을 대시보드(W-40) 카운터로 상시 감시할 것.
- **규제 유예 창 모니터링 (전 여정 · W-42·W-47)**: 선불업 등록 유예는 '회원간 송금 금지·용도 제한·수수료 0'을 지키는 한에서만 유효하다. 외부 계좌 유인 신고(W-44)·수수료 0 위반·용도 이탈을 리포트(W-47) KPI로 상시 추적해 유예 조건 이탈을 조기 경보할 것.
- **원장의 부원 노출 깊이 (⑤·⑥ · W-25)**: 자동 정산·환급의 신뢰는 부원이 원장을 '스스로 검증'할 수 있을 때 완성된다. W-25 지갑·마이에서 부원이 자기 자금 이동(예치→출금→환급)을 얼마나 투명하게 열람하는지에 따라 재예치(리텐션)가 갈리므로, 개인 원장 뷰의 가독성을 리텐션 지표와 연결해 검증할 것.
- **소셜 신뢰 신호의 조기 확보 (소셜 · W-27·W-33)**: 핵심문제 4(친분 의존)를 푸는 축은 쇼 피드의 활동·후기 축적이다. 초기 콜드스타트(후기 0건) 구간에서 신뢰 신호를 어떻게 부트스트랩할지(운영자 큐레이션·완료 차수 자동 요약 게시 등)를 별도 과제로 다룰 것.
